# Sufficiency Gate 改进总结

## 背景

原 `sufficiency.py` 存在两个问题：
1. **强制设置用户概念**：`topic_query_source` 默认强制为 "user_query"，不够灵活
2. **直接替换大模型回复**：当证据不足时，直接拒绝回答或强制澄清，跳过了大模型

## 改进方案：后置建议模式

### 核心变化

```
旧流程：Sufficiency Gate → 决定是否回答 → 强制替换/拒绝回答
新流程：大模型正常回答 → Sufficiency Gate 评估 → 在回答后追加建议
```

### 主要修改

#### 1. `app/sufficiency.py`

- **移除强制拦截逻辑**：`decision` 不再返回 "refuse" 或 "clarify"，默认都是 "answer"
- **新增 `advice` 字段**：包含建议类型、消息、建议操作
- **新增 `format_advice_for_answer()` 函数**：将建议格式化为可追加的文本
- **保留诊断信息**：`evidence_quality`, `confidence_level`, `missing_aspects` 等仍保留

#### 2. `app/qa.py`

- **导入新函数**：`from app.sufficiency import format_advice_for_answer`
- **调整流程**：
  - 大模型先生成回答（`_build_answer`）
  - 然后追加 sufficiency gate 的建议
- **移除强制拦截分支**：原 `elif decision == "clarify":` 和 `else:`（refuse）分支已移除

#### 3. `app/config.py`

- **新增配置项**：`sufficiency_append_advice: bool = True`
  - 控制是否在回答后追加证据质量建议

## 新工作流程

### 正常流程（证据充足）

```
用户：请介绍 Transformer 的架构
AI：Transformer 架构由注意力机制和前馈网络组成...
[无额外提示]
```

### 证据有限（1条）

```
用户：请介绍 Transformer 的架构
AI：Transformer 架构由注意力机制和前馈网络组成...

==================================================
📋 回答质量提示
==================================================
⚠️ 当前回答基于有限的证据（仅1条），建议谨慎参考。

💡 建议:
  1. 补充更多相关论文或实验证据
  2. 明确更具体的问题范围

📊 证据概况:
  - 证据数量: 1
  - 证据质量: low
  - 置信度: low
==================================================
```

### 无证据

```
用户：比特币今天价格是多少？
AI：[大模型基于一般知识回答]...

==================================================
📋 回答质量提示
==================================================
⚠️ 当前问题在知识库中未找到相关证据。
以下回答基于模型的一般知识，可能不够准确。

💡 建议:
  1. 补充论文标题、作者、年份或会议等线索
  2. 确认知识库是否包含相关主题
==================================================
```

## 优势对比

| 方面 | 旧方案 | 新方案 |
|------|--------|--------|
| **用户体验** | 直接拒绝，看不到任何内容 | 能看到回答，同时了解质量 |
| **信息传递** | 只有负面信息 | 有回答 + 质量提示 |
| **灵活性** | 强制拦截 | 用户可自行判断 |
| **大模型价值** | 被跳过 | 充分发挥 |
| **适用场景** | 严格场景 | 灵活场景 |

## 配置选项

在 `config.yaml` 中：

```yaml
sufficiency_gate_enabled: true      # 启用 Sufficiency Gate
sufficiency_append_advice: true     # 是否在回答后追加建议
```

## 演示

运行演示脚本：

```bash
python demo_sufficiency_advisory.py
```

## 向后兼容

- 所有现有配置仍然有效
- `sufficiency_gate_enabled` 仍可关闭整个功能
- 新增配置 `sufficiency_append_advice` 默认为 `True`

## 测试

关键测试点：
1. ✅ 证据充足时不显示提示
2. ✅ 证据有限时显示谨慎提示
3. ✅ 无证据时显示警告提示
4. ✅ 只有噪声证据时显示警告
5. ✅ 主题不匹配时显示警告
6. ✅ 配置 `sufficiency_append_advice: false` 时不追加提示
