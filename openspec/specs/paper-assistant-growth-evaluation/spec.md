# paper-assistant-growth-evaluation 规范

## 目的
待定 - 由归档变更 improve-paper-assistant-friendly-mode 创建。归档后请更新目的。
## 需求
### 需求:系统必须维护增长优先评测问题集
系统必须维护一个版本化的论文助理评测问题集，且每条样本必须包含可用于分桶统计与多轮重放的结构化字段。

#### 场景:问题集结构完整
- **当** 系统加载增长评测问题集
- **那么** 每条样本必须至少包含 `id`、`session_id`、`turn_index`、`question`、`bucket`、`same_topic`、`is_out_of_corpus` 字段

#### 场景:问题集分桶覆盖
- **当** 系统执行发布评测
- **那么** 问题集必须覆盖 `A_open_summary`、`B_multi_turn`、`C_control_mixed`、`D_ooc` 四个桶

### 需求:系统必须执行旧新双策略对比评测
系统必须在同一问题集和一致配置下分别执行旧策略与新策略，并输出可复现的对比结果。

#### 场景:双策略配置一致
- **当** 系统执行双策略回放
- **那么** 除 `assistant_mode_force_legacy_gate` 外，其它关键开关必须一致，且报告必须记录配置快照

#### 场景:统一统计口径
- **当** 系统汇总评测指标
- **那么** 统计必须基于 `qa_report.decision` 与对应字段，不得混用多种决策口径

### 需求:系统必须应用增长优先放行与护栏
系统必须以“可追溯回答率增长”为优先目标，同时满足语料外与连续澄清等风险护栏。

#### 场景:增长优先放行
- **当** 系统计算发布结果
- **那么** 新策略在可追溯回答率上必须相对旧策略取得显著提升，且拒答率不高于约定阈值

#### 场景:硬护栏触发回滚
- **当** 语料外误答率或连续澄清超阈值
- **那么** 系统必须标记本次评测不通过，并给出回滚建议
