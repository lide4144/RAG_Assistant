## 需求
### 需求:系统必须将论文导入纳入统一任务状态机
系统必须将浏览器侧论文导入任务纳入与图构建一致的统一任务状态机，对外使用相同的 `queued`、`running`、`succeeded`、`failed`、`cancelled` 状态集合与任务查询接口。

#### 场景:论文导入任务被受理
- **当** 用户通过前端上传文件并触发论文导入
- **那么** 系统必须为该导入生成 `taskId`、`taskKind=library_import`，并通过统一任务查询接口返回当前状态

#### 场景:论文导入任务失败
- **当** 论文导入在解析、清洗或索引阶段失败
- **那么** 系统必须将任务状态置为 `failed`，并返回失败阶段、错误摘要与恢复建议供前端展示

### 需求:系统必须提供统一任务状态机
系统必须为后台任务提供统一状态机，至少包含 `idle`、`queued`、`running`、`succeeded`、`failed`、`cancelled`，禁止各任务使用互不兼容的私有状态集合。该状态机必须覆盖图构建、论文导入、聊天问答与规划问答四类后台任务，并且每个任务都必须拥有稳定的任务标识用于订阅、恢复与取消。

#### 场景:任务从排队到完成
- **当** 用户触发图构建任务、论文导入任务、聊天问答或规划问答且任务被系统受理
- **那么** 系统必须按 `queued -> running -> succeeded/failed/cancelled` 的状态序列对外报告任务状态

### 需求:系统必须提供结构化任务进度事件
系统必须输出结构化进度事件，至少包含任务标识、任务类型、`stage`、`processed`、`total`、`elapsedMs` 与 `message` 字段，禁止仅输出不可解析的纯文本日志作为唯一进度来源。对于论文导入任务，结构化进度事件还必须支持批次级统计与当前处理项信息；对于聊天与规划任务，结构化进度事件还必须支持回答增量、来源集合、错误信息与完成事件的顺序化表达。

#### 场景:实体抽取阶段进度可见
- **当** 图构建进入实体抽取阶段
- **那么** 系统必须持续输出 `processed/total` 进度并允许前端渲染为可见进度条

#### 场景:论文导入批次进度可见
- **当** 论文导入任务持续运行超过一次轮询周期
- **那么** 系统必须输出随时间推进的批次统计或阶段内统计，避免前端只能渲染长时间静止的“处理中”

#### 场景:聊天任务事件可回放
- **当** 聊天或规划任务已经产生部分回答增量且客户端随后重新连接
- **那么** 系统必须能够按事件顺序返回此前已经产生的结构化事件，使前端恢复到断连前的可见进度

### 需求:系统必须为论文导入任务输出批次级结构化进度统计
系统必须为 `library_import` 任务输出可直接渲染批次进度的结构化统计，至少包括批次总数、已完成数、处理中数、失败数与当前处理项；禁止要求前端仅根据阶段名称或固定权重推导总体完成度。

#### 场景:前端渲染真实批次进度
- **当** 前端轮询论文导入任务状态
- **那么** 系统必须返回足以渲染真实批次进度的数据，使页面能够显示“已完成 X/Y 篇、失败 Z 篇、当前处理 A.pdf”之类的反馈

### 需求:系统必须允许任务进度事件携带逐项状态摘要
系统必须允许长任务进度事件附带有限数量的逐项状态摘要，用于前端展示最近处理项、失败项或当前卡住项；禁止把所有细节都压缩成单条不可分解的 message 文本。

#### 场景:批量导入中展示失败样本
- **当** 一次论文导入中已有若干篇失败
- **那么** 任务状态输出必须能够返回这些失败样本的名称与错误摘要，供前端在进度面板中直接展示

### 需求:系统必须提供任务失败可恢复反馈
系统必须在任务失败时输出可恢复错误信息，至少包含失败阶段、错误摘要与可执行下一步动作（重试或查看详情）。

#### 场景:图构建中途失败
- **当** 图构建在任一阶段发生异常
- **那么** 系统必须将任务状态置为 `failed`，并返回失败阶段与错误摘要供前端展示

### 需求:系统必须为后台任务提供恢复查询入口
系统必须为统一任务状态机增加面向后台恢复的查询入口，允许前端按 `taskId/jobId` 获取最新状态、结果摘要与最近事件游标。该入口必须适用于图构建、论文导入以及后续纳入状态机的聊天/规划任务，禁止仅支持“当前页面轮询时碰巧能看到”的临时状态。

#### 场景:页面刷新后重新附着任务状态
- **当** 用户刷新页面后仍有图构建或论文导入任务处于活动状态
- **那么** 系统必须允许前端基于既有任务标识恢复该任务的当前状态与最新进度，而不是把任务视为丢失
