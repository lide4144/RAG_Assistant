## 需求
### 需求:系统必须提供面向科研用户的三分区工作台导航
系统必须提供 `Library`、`Chat`、`Ideas` 三个一级工作区，并且默认进入 `Chat` 以支持快速提问。系统禁止将检索参数与调试字段作为默认首屏信息。

#### 场景:首屏进入可提问状态
- **当** 用户打开系统
- **那么** 界面必须在 1 次可见交互内提供提问输入框与当前专题上下文

### 需求:系统必须支持从回答到灵感沉淀的一键闭环
系统必须允许用户在回答完成后一键生成灵感卡片草稿，并在同一流程中完成保存。

#### 场景:回答后可直接生成卡片
- **当** 助手回答完成且存在可追溯证据
- **那么** 界面必须提供“生成灵感卡片”操作，并生成可编辑草稿

### 需求:系统必须提供任务导向术语
前台用户文案必须使用任务导向术语（例如导入论文、证据来源、灵感卡片），禁止默认展示 ingest/index/retrieval 等工程术语。

#### 场景:用户界面术语符合任务语义
- **当** 用户浏览主流程页面
- **那么** 页面中的主操作与提示必须采用任务导向术语

### 需求:长耗时任务必须提供可观察进度
系统必须在导入、解析、索引等长耗时本地任务中提供实时进度反馈，且必须支持在统一任务中心按任务查看阶段状态、失败原因与重试入口，禁止仅显示无限期等待状态。

#### 场景:导入流程显示阶段进度
- **当** 用户发起论文导入并进入多阶段处理
- **那么** 界面必须持续显示阶段进度与当前处理状态

#### 场景:失败任务可在任务中心恢复
- **当** 任一长任务执行失败
- **那么** 任务中心必须显示失败阶段与错误摘要，并提供可执行的重试入口

### 需求:推理失败必须可恢复且不破坏会话
系统必须捕获推理与外部模型调用异常并以用户可理解方式提示，禁止因单次失败导致界面崩溃或会话重置。

#### 场景:网络波动导致模型调用超时
- **当** 本地应用调用外部模型发生超时或断连
- **那么** 界面必须显示可恢复错误提示并保留当前历史对话内容

### 需求:UI 会话状态必须控制内存体积
系统必须将 `session_state` 限制为渲染必需轻量字段，重型 trace 与检索中间结果必须落盘并按需懒加载。

#### 场景:多轮对话后的状态体积受控
- **当** 用户连续进行多轮问答
- **那么** 系统内存中的会话状态必须不随历史 trace 线性膨胀且可按需查看磁盘明细

### 需求:系统必须提供产品化工作台入口
研究助手工作台必须支持产品化聊天入口与任务上下文延续，禁止仅依赖 Streamlit 单页作为唯一入口。

#### 场景:工作台从产品界面进入
- **当** 用户在新前端进入研究助手模式
- **那么** 系统必须复用历史会话上下文并提供连续追问能力

### 需求:系统必须保留研究流程可追溯性
工作台必须保留关键过程可追溯信息（query 演化、证据来源、决策原因），但必须以可折叠或按需展开方式呈现。

#### 场景:按需展开决策依据
- **当** 用户主动查看“为何这样回答”
- **那么** 系统必须展示该轮 query 演化与证据决策摘要

### 需求:系统必须提供 Pipeline Workbench 一级入口
系统必须在产品化工作台中提供 `Pipeline` 一级入口，用于承载导入、处理与建图任务，禁止仅通过聊天页面承载全部任务操作。

#### 场景:用户可从工作台进入处理流程
- **当** 用户进入研究助手工作台
- **那么** 界面必须可在一级导航中访问 Pipeline 区域并查看当前任务状态

### 需求:系统必须在工作台展示长任务实时状态
系统必须在 Pipeline 区域持续展示长任务的阶段状态、进度与最近更新时间，禁止仅显示无限等待态。

#### 场景:图构建运行中可见状态
- **当** 图构建任务处于 `running`
- **那么** Pipeline 区域必须显示当前阶段、进度值与累计耗时
