# frontend-pipeline-ops-dashboard 规范

## 目的
定义知识处理页的运行态展示、任务事件连接和导入风险反馈，确保管理员能够在同一页面理解流水线状态、产物健康与降级结果。

## 需求
### 需求:系统必须提供知识库构建流水线的统计总览区
系统必须在页面顶部提供 4 个关键统计卡片，关键数值必须使用等宽数字排版（tabular-nums）并以高视觉优先级字号展示，禁止将数字与说明以同级弱对比呈现；当存在降级运行、缺失产物或待处理积压时，统计区必须以高可见度摘要方式提示风险。

#### 场景:进入知识处理页
- **当** 用户进入“知识处理”页面
- **那么** 页面必须显示具备高对比数字焦点的统计总览区

#### 场景:总览区包含降级风险
- **当** 最近一次导入存在降级结果或产物缺失
- **那么** 总览区必须显示风险摘要卡片或徽标，并可导航到详细原因或产物管理视图

### 需求:系统必须提供动态数字反馈
系统必须为关键数字指标提供动态过渡展示，禁止在数据刷新时出现无过渡跳变。

#### 场景:统计指标更新
- **当** 流水线状态变化导致指标刷新
- **那么** 指标数字必须以可感知的过渡动画更新

### 需求:系统必须提供四阶段实时状态指示
系统必须将 Import、Clean、Index、Graph Build 四阶段重构为单一横向步骤条并使用连接线表达流程连续性。系统必须通过 `StatusMapper` 映射阶段状态，且至少覆盖 `not_started`、`processing`、`unknown`、`degraded`、`failed_with_fallback`；当某阶段已降级完成时，系统必须在步骤条上展示可辨识的降级语义标签，而不是仍显示为成功完成。

#### 场景:步骤条状态映射
- **当** 阶段状态更新
- **那么** 系统必须在步骤条上展示映射后的语义标签和图标，而非原始枚举文本

#### 场景:导入阶段发生降级
- **当** Import 阶段因 Marker LLM 失败而回退到非 LLM 路径并继续执行
- **那么** 系统必须在步骤条上将该阶段显示为降级完成，并提供可展开的原因说明

### 需求:系统必须在存在图构建任务时展示终端风格日志区
系统必须将 Graph Build 日志展示为终端风格控制台，包含深色背景、等宽字体和可读性优化滚动条，禁止使用普通正文样式渲染高频日志流。

#### 场景:Graph Build 日志持续输出
- **当** 图构建任务运行中并持续返回日志
- **那么** 日志区域必须以终端风格持续追加并保持可滚动浏览

### 需求:系统必须提供导入历史标识可读化能力
系统必须对导入记录中的长运行 ID 进行截断展示（保留前 8 位），并必须提供完整 ID 的悬浮提示与一键复制操作。

#### 场景:查看导入历史表格
- **当** 用户浏览 `Run` 列
- **那么** 系统必须显示短 ID、悬浮可见完整值且支持点击复制

### 需求:系统必须以部署友好策略建立任务事件连接
知识处理页必须以部署友好策略建立任务事件 WebSocket 连接。未显式配置 `NEXT_PUBLIC_GATEWAY_WS_URL` 时，系统必须根据当前页面来源自动推导 `/ws` 地址，并在 HTTPS 页面使用 `wss`；禁止默认连接 `ws://localhost:8080/ws`。

#### 场景:远程部署下进入知识处理页
- **当** 用户通过服务器域名、公网 IP 或平台单入口域名进入“知识处理”页面且未显式配置 `NEXT_PUBLIC_GATEWAY_WS_URL`
- **那么** 页面必须尝试连接当前站点来源对应的 `/ws`，使任务状态与进度事件可在远程部署中正常建立
