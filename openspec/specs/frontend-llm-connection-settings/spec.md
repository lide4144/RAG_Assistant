# frontend-llm-connection-settings 规范

## 目的
待定 - 由归档变更 add-llm-visual-config-and-model-detection 创建。归档后请更新目的。
## 需求
### 需求:系统必须提供 LLM 连接设置面板
系统必须在前端提供 stage 化设置面板，不仅包含基础连接参数，还必须支持 `answer`、`embedding`、`rerank` 三路独立配置输入，禁止仅提供单路配置表单。

#### 场景:管理员填写连接参数
- **当** 管理员进入设置面板
- **那么** 系统必须显示 answer、embedding、rerank 三组配置字段并允许独立编辑

### 需求:系统必须在探测成功后提供模型选择
系统必须在模型探测成功后展示模型下拉框，并禁止在未探测成功时展示无效模型选项。

#### 场景:探测成功
- **当** 后端返回至少一个可用模型
- **那么** 系统必须显示模型下拉并允许管理员选择目标模型

### 需求:系统必须支持保存最终配置
系统必须提供保存按钮，将三路配置一次性提交到后端配置持久化接口。系统必须保持对旧单路接口响应的兼容处理，避免 UI 崩溃。

#### 场景:完成配置保存
- **当** 管理员完成三路配置并点击保存
- **那么** 系统必须提交三路配置并反馈保存成功状态

### 需求:系统必须提供可理解的错误反馈
系统必须根据后端错误码展示可理解提示，禁止仅显示无上下文的通用失败文案。

#### 场景:鉴权失败提示
- **当** 后端返回 `AUTH_FAILED`
- **那么** 系统必须提示管理员检查 API Key 或权限

### 需求:系统必须提供阶段级错误反馈
系统必须在保存失败时明确提示是哪个 stage 的配置失败，并给出可理解的错误原因，禁止仅显示无上下文的通用失败文案。

#### 场景:单 stage 校验失败
- **当** 仅 `embedding` 配置缺失关键字段
- **那么** 系统必须提示 embedding 配置错误且保留 answer/rerank 输入状态
