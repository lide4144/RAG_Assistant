# local-llm-bootstrap-and-defaults 规范

## 目的
定义本地 LLM 安装、初始化和默认模型档位，确保 embedding 与 rewrite 能以本地主路径稳定启用，同时为 rerank 保留兼容的远端路径。

## 需求
### 需求:系统必须提供本地模型安装与初始化基线
系统必须提供可复现的本地模型安装与初始化流程，并必须以 Ollama 作为默认主路径。流程必须覆盖 `embedding`、`rewrite` 两段模型的安装准备、启动校验与常见故障排查，并明确 `rerank` 默认保留外部 API / vLLM 兼容路径。

#### 场景:开发机按文档完成本地模型初始化
- **当** 开发者在 8GB 显存设备上按文档执行安装与初始化步骤
- **那么** 系统必须能够完成 `embedding` 与 `rewrite` 的本地可用性检查，并输出明确的成功或失败信息

### 需求:系统必须定义本地模型默认档位与降级备选
系统必须内置本地模型默认值与降级备选：`embedding` 默认 `bge-m3`、备选 `nomic-embed-text`；`rewrite` 默认 `qwen2.5:3b`、备选 `qwen2.5:1.5b`。系统必须明确 `rerank` 默认不纳入 Ollama 主路径，而保留外部 API 或其他 OpenAI-compatible 路径。

#### 场景:默认档位不可用时切换备选
- **当** 默认模型拉取失败或运行不稳定
- **那么** 系统必须允许切换到对应备选模型并继续完成初始化流程，且不得要求 `rerank` 同步切换到本地 Ollama
