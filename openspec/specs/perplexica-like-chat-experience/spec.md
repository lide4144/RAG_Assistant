## 需求
### 需求:系统必须提供产品化聊天主界面
系统必须提供面向最终用户的聊天主界面，界面必须支持多轮消息、流式输出、模式切换与来源卡片展示；禁止仅以开发者审查字段作为主界面交互。

#### 场景:首轮对话可流式呈现
- **当** 用户提交首个问题
- **那么** 系统必须在主界面逐段显示回答内容，并在回答完成后展示来源摘要卡片

### 需求:系统必须提供最小模式切换能力
系统必须提供 `Local`、`Web`、`Hybrid` 三种模式入口，并在每轮请求中明确记录用户选择的模式。

#### 场景:用户切换到 Web 模式
- **当** 用户将模式切换为 `Web` 并发起问题
- **那么** 请求上下文必须携带 `Web` 模式标记，且回答来源必须可展示 Web 来源
