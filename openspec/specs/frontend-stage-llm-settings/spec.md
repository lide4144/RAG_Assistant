# frontend-stage-llm-settings 规范

## 目的
待定 - 由归档变更 frontend-multi-stage-llm-settings 创建。归档后请更新目的。
## 需求
### 需求:前端必须提供三路 Stage LLM 设置面板
系统必须为辅助模型节点提供“继承全局配置”能力。对于 Rewrite、Graph Entity 等辅助节点，默认必须隐藏 `api_base` 与 `api_key` 输入框并显示“🔄 已继承全局配置”。仅当用户显式开启 `独立配置 (Override)` 时，系统才可展开并允许编辑独立凭据字段。

#### 场景:辅助模型默认继承
- **当** 用户进入模型设置页且未开启 Override
- **那么** 辅助模型必须隐藏独立凭据输入并显示继承提示

#### 场景:启用 Override
- **当** 用户开启辅助模型 Override
- **那么** 系统必须展开该模型独立 `api_base` 和 `api_key` 字段

### 需求:前端必须提交三路配置载荷
当管理员保存配置时，前端必须提交包含 answer/embedding/rerank 三路对象的载荷到管理接口，禁止仅提交单一路径字段。

#### 场景:三路配置保存
- **当** 管理员完成三路配置并点击保存
- **那么** 系统必须向 `/api/admin/llm-config` 发送三路结构请求并展示保存结果

### 需求:前端必须正确回显三路持久化配置
设置页在加载 `/api/admin/llm-config`、`/api/admin/pipeline-config`、`/api/admin/runtime-overview` 时，必须对非 JSON 与错误响应进行受控处理；系统必须先判定响应状态与类型再解析载荷，避免错误页触发 JSON 解析崩溃。

#### 场景:管理接口返回 404 HTML
- **当** 管理接口返回状态码 `4xx/5xx` 且 `content-type` 为 `text/html`
- **那么** 页面必须进入可读错误状态并保持设置表单可继续编辑，不得抛出 `Unexpected token '<'`

#### 场景:管理接口返回结构化错误 JSON
- **当** 管理接口返回错误 JSON（含 `detail` 等字段）
- **那么** 系统必须优先展示可执行错误提示，而不是显示通用解析失败信息

### 需求:前端必须提供字段级未保存状态提示
前端必须在用户修改但尚未保存的字段标签旁显示柔和的“⚠️ 未保存”徽章，并必须在保存成功后清除该标记。

#### 场景:修改字段后未保存
- **当** 用户修改任一模型配置字段但未点击保存
- **那么** 对应字段标签必须显示“⚠️ 未保存”提示
