# frontend-stage-llm-settings 规范

## 目的
定义前端多位点 Stage LLM 设置面板的组织方式、载荷约束和高级调优信息展示，确保高频配置与底层参数能在同一页面中有序共存。
## 需求
### 需求:前端必须提供多位点 Stage LLM 设置面板
系统必须在单一管理面板中提供可扩展的 stage 配置能力，除 `answer`、`embedding`、`rerank` 外必须支持 `rewrite` 与 `graph_entity` 位点。系统必须为辅助模型节点提供“继承全局配置”能力，并将多位点详细配置置于核心模型选择区之后。对于 Rewrite、Graph Entity 等辅助节点，默认必须隐藏 `api_base` 与 `api_key` 输入框并显示“🔄 已继承全局配置”。仅当用户显式开启 `独立配置 (Override)` 时，系统才可展开并允许编辑独立凭据字段。

#### 场景:辅助模型默认继承
- **当** 用户进入模型设置页且未开启 Override
- **那么** 辅助模型必须隐藏独立凭据输入并显示继承提示

#### 场景:启用 Override
- **当** 用户开启辅助模型 Override
- **那么** 系统必须展开该模型独立 `api_base` 和 `api_key` 字段

#### 场景:查看多位点配置
- **当** 管理员继续向下浏览设置页
- **那么** 页面必须仍可编辑各 stage 的详细字段，并保留继承与 Override 行为

#### 场景:单位点校验失败
- **当** 仅 `rewrite` 配置缺失必填模型
- **那么** 系统必须仅提示 `rewrite` 位点错误，并保留其他位点已填写内容

### 需求:前端必须提交多位点配置载荷
当管理员保存配置时，前端必须提交包含 `answer`、`embedding`、`rerank`、`rewrite`、`graph_entity` 五个位点对象的载荷到管理接口，并兼容旧三路读取结果，禁止仅提交单一路径字段。

#### 场景:保存全模型配置
- **当** 管理员完成全模型编辑并点击保存
- **那么** 系统必须向 `/api/admin/llm-config` 发送全量位点结构请求并展示保存结果

### 需求:前端必须正确回显三路持久化配置
设置页在加载 `/api/admin/llm-config`、`/api/admin/pipeline-config`、`/api/admin/runtime-overview` 时，必须对非 JSON 与错误响应进行受控处理；系统必须先判定响应状态与类型再解析载荷，避免错误页触发 JSON 解析崩溃。设置页还必须利用运行态或任务接口提供的活动任务锁信息，在有活动任务时切换为只读展示而不是继续允许编辑。

#### 场景:管理接口返回 404 HTML
- **当** 管理接口返回状态码 `4xx/5xx` 且 `content-type` 为 `text/html`
- **那么** 页面必须进入可读错误状态并保持设置表单可继续编辑，不得抛出 `Unexpected token '<'`

#### 场景:管理接口返回结构化错误 JSON
- **当** 管理接口返回错误 JSON（含 `detail` 等字段）
- **那么** 系统必须优先展示可执行错误提示，而不是显示通用解析失败信息

#### 场景:运行中任务导致设置只读
- **当** 运行态或任务接口返回存在活动任务占用设置锁
- **那么** 页面必须切换到只读状态并展示导致锁定的任务摘要

### 需求:前端必须提供字段级未保存状态提示
前端必须在用户修改但尚未保存的字段标签旁显示柔和的“⚠️ 未保存”徽章，并必须在保存成功后清除该标记。

#### 场景:修改字段后未保存
- **当** 用户修改任一模型配置字段但未点击保存
- **那么** 对应字段标签必须显示“⚠️ 未保存”提示

### 需求:前端必须将高级调优参数默认折叠
前端必须将 Marker Runtime Tuning 相关底层参数收纳到默认折叠的“高级模型调优”面板中，禁止首次进入页面时直接将全部批处理参数暴露在主要操作区域。

#### 场景:首次进入设置页
- **当** 用户首次打开“模型设置”页面
- **那么** 高级模型调优面板必须默认处于折叠状态

### 需求:前端必须为晦涩参数提供中文解释
前端必须在识别批量、检测批量、布局批量、OCR 错误批量、表格识别批量和精度类型等参数旁提供 Tooltip 或等效帮助提示，解释其作用与影响。

#### 场景:查看高级参数标签
- **当** 用户查看高级模型调优面板内的参数标签
- **那么** 页面必须提供可见的帮助提示入口，并说明该参数的中文含义

### 需求:前端必须在活动任务期间锁定设置编辑
当前存在活动后台任务时，前端模型设置页必须进入只读模式，并明确展示占用该锁的活动任务摘要。只读模式下必须禁止编辑字段、触发检测与提交保存，防止用户在任务执行期间误以为配置已即时生效。

#### 场景:运行中的任务锁定设置页
- **当** 用户打开模型设置页且系统存在 `queued` 或 `running` 的后台任务
- **那么** 页面必须显示设置已锁定的提示，并禁用所有会改变运行时配置的输入与操作按钮

### 需求:后端必须拒绝活动任务期间的设置保存
当存在活动后台任务时，设置保存接口必须返回结构化拒绝结果，禁止仅依赖前端禁用按钮来防止配置变更。拒绝结果必须包含机器可读错误码与活动任务摘要，便于前端展示明确原因。

#### 场景:运行中任务触发保存被拒绝
- **当** 客户端在活动任务期间向任一设置保存接口提交配置变更
- **那么** 系统必须返回拒绝保存的结构化错误，说明当前有活动任务占用设置锁
