## 需求
### 需求:系统必须提供可检索的论文目录管理页
系统必须在 Library 中提供论文目录管理视图，支持按标题、来源、专题与状态进行检索和筛选。

#### 场景:用户按关键词定位论文
- **当** 用户在目录页输入关键词
- **那么** 系统必须返回标题或来源匹配的论文列表

#### 场景:用户按专题筛选论文
- **当** 用户选择某专题筛选
- **那么** 系统必须仅展示属于该专题范围的论文

### 需求:系统必须展示论文可维护元数据
目录页必须展示每篇论文的稳定标识、标题、来源类型、导入时间与当前状态，支持用户判断是否重复导入或待处理。

#### 场景:用户查看论文详情
- **当** 用户打开某论文详情
- **那么** 系统必须展示核心元数据与专题归属信息

### 需求:系统必须支持目录级专题批量操作
系统必须允许用户在目录页对多篇论文进行专题归类或移除操作，并保持与 Chat 侧 topic scope 一致。

#### 场景:批量加入专题
- **当** 用户选择多篇论文并指定专题
- **那么** 系统必须一次性更新专题映射并反馈变更数量
