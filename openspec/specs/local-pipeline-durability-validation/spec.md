## 需求
### 需求:关键产物写入必须具备原子性
系统必须将关键产物写入到同目录临时文件并在写入完成后进行原子替换，禁止直接覆盖目标文件。

#### 场景:写入成功后原子替换
- **当** 系统完成 `chunks.jsonl` 或 `papers.json` 的全部写入
- **那么** 目标文件必须仅在完整写入后可见，且不出现半成品内容

#### 场景:写入中断后的临时文件清理
- **当** 写入过程发生异常或进程中断
- **那么** 系统必须保留上一版可用目标文件并清理残留临时文件

### 需求:本地并发写入必须受互斥保护
系统必须在落盘与索引更新阶段使用进程级文件锁，禁止同一资源被并发写入。

#### 场景:双实例同时触发导入
- **当** 两个本地实例同时对同一数据目录执行导入或索引更新
- **那么** 系统必须仅允许一个实例进入临界区，另一个实例必须等待或失败并返回可理解提示

### 需求:大文件处理必须限制内存峰值
系统必须对大体量 JSON/JSONL 采用流式写入与增量处理，禁止在内存中构建完整数据副本。

#### 场景:百万级记录写入
- **当** 输入记录达到百万级别
- **那么** 系统必须以流式方式写入并保持内存占用在可配置上限内

### 需求:错误密集数据必须支持快速失败
系统必须对校验错误数量设置上限并在达到阈值后终止校验，禁止无限累计错误导致质检进程失稳。

#### 场景:错误数量超过阈值
- **当** 校验过程中错误数量达到 `max_errors`
- **那么** 系统必须立即停止继续校验并返回包含截断提示的错误摘要
