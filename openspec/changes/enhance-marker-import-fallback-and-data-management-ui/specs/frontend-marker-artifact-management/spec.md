## 新增需求

### 需求:系统必须提供 Marker 产物管理视图
系统必须提供独立的 Marker 产物管理视图，集中展示 `data/indexes` 与 `data/processed` 下的核心产物、来源阶段、更新时间与健康状态，禁止要求管理员通过服务器文件系统手动确认产物是否存在。

#### 场景:进入产物管理视图
- **当** 管理员打开 Marker 产物管理页面或面板
- **那么** 系统必须按 `indexes` 与 `processed` 分组展示可管理产物，并显示每个产物的名称、类型、更新时间与状态

### 需求:系统必须提供产物级操作与风险提示
系统必须允许管理员对既有 `Index` 与 `processed` 产物执行受控操作，至少包括查看详情、复制路径、重新生成入口与删除前风险确认；系统禁止在无确认的情况下直接执行破坏性删除。

#### 场景:管理员准备删除产物
- **当** 管理员在产物管理视图中触发删除操作
- **那么** 系统必须展示受影响阶段、可能后果与确认步骤，并在确认前阻止实际删除

### 需求:系统必须将产物状态与流水线状态关联展示
系统必须将产物可用性与 Import、Clean、Index、Graph Build 等流水线阶段关联展示，使管理员能够从同一视图判断缺失产物是否导致当前阶段降级、阻塞或需要重建。

#### 场景:存在缺失或过期产物
- **当** 某个 `processed` 或 `indexes` 产物缺失、过期或与最近一次运行不一致
- **那么** 系统必须显示关联阶段、影响说明与建议动作，而不是仅显示“文件不存在”
