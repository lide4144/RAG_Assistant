## 修改需求

### 需求:系统必须将产物状态与流水线状态关联展示
系统必须将产物可用性与 Import、Clean、Index、Graph Build 等流水线阶段关联展示，使管理员能够从同一视图判断缺失产物是否导致当前阶段降级、阻塞或需要重建。系统必须按产物所属阶段的最近完成时间判定 stale，禁止使用单一全局“最近一次运行时间”把同轮导入内较早生成的健康产物误判为待更新。

#### 场景:存在缺失或过期产物
- **当** 某个 `processed` 或 `indexes` 产物缺失、过期或与所属阶段最近一次完成时间不一致
- **那么** 系统必须显示关联阶段、影响说明与建议动作，而不是仅显示“文件不存在”

#### 场景:同轮导入后查看产物
- **当** 管理员查看一次刚完成导入生成的 `chunks`、`papers`、`bm25` 或 `vec` 产物，且这些文件时间不早于各自所属阶段的最近完成时间
- **那么** 系统必须将这些产物显示为可用，禁止仅因导入汇总状态在更晚时间写入就标记为待更新
