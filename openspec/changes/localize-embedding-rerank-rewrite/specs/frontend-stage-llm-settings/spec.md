## 修改需求

### 需求:前端必须提供多位点 Stage LLM 设置面板
系统必须在单一管理面板中提供可扩展的 stage 配置能力，除 `answer`、`embedding`、`rerank` 外必须支持 `rewrite` 与 `graph_entity` 位点。每个位点必须提供独立字段校验与错误提示，禁止一个位点的校验错误阻断其他位点输入保留。

#### 场景:单位点校验失败
- **当** 仅 `rewrite` 配置缺失必填模型
- **那么** 系统必须仅提示 `rewrite` 位点错误，并保留其他位点已填写内容

### 需求:前端必须提交多位点配置载荷
系统在保存配置时必须提交包含全模型位点对象的结构化载荷，并必须兼容旧三段读取结果，禁止因历史数据结构导致页面崩溃。

#### 场景:保存全模型配置
- **当** 管理员完成全模型编辑并点击保存
- **那么** 系统必须提交全量位点配置并在保存成功后可正确回显
