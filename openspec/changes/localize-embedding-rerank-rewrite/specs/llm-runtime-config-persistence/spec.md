## 修改需求

### 需求:系统必须持久化 LLM 路由配置
系统必须提供持久化结构保存管理员选择的路由配置，并支持读取最新生效配置。持久化配置必须覆盖 `answer`、`embedding`、`rerank`、`rewrite` 与 `graph_entity` 位点，每个位点至少包含 `provider`、`api_base`、`api_key`、`model`。系统必须保持旧格式载荷兼容并完成映射。

#### 场景:保存并读取全模型配置
- **当** 管理员保存全模型位点配置
- **那么** 系统必须在后续读取中返回完整位点结构且字段语义一致

### 需求:系统必须消除 rewrite 对 answer 的隐式覆盖
系统必须禁止在配置加载阶段强制使用 `answer` 路由覆盖 `rewrite` 路由。系统必须将 `rewrite` 视为独立位点并单独生效。

#### 场景:rewrite 与 answer 使用不同模型
- **当** 管理员将 `rewrite` 与 `answer` 配置为不同模型
- **那么** 系统必须按各自位点配置生效，且不得在加载时将 rewrite 覆盖为 answer
