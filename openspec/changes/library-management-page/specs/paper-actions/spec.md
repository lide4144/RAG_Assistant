## 新增需求

### 需求:查看论文详情
系统必须支持查看单篇论文的详细信息。

#### 场景:打开详情弹窗
- **当** 用户点击论文卡片上的"查看"按钮
- **那么** 系统弹出论文详情模态框
- **并且** 显示论文 ID、标题、来源、状态、导入时间、专题列表
- **并且** 显示论文各处理阶段的状态（import, parse, clean, index, graph）

### 需求:删除单篇论文
系统必须支持删除单篇论文。

#### 场景:删除论文确认
- **当** 用户点击论文卡片上的"删除"按钮
- **那么** 系统显示确认对话框
- **并且** 对话框列出将被删除的内容：元数据、向量索引、专题关联、处理产物

#### 场景:执行删除操作
- **当** 用户确认删除
- **那么** 系统调用 POST /api/library/papers/{id}/delete
- **并且** 成功后从列表移除该论文卡片
- **并且** 显示 toast 提示"已删除"
- **并且** 提示用户"建议重建索引"

### 需求:重建论文索引
系统必须支持重建单篇论文的向量索引。

#### 场景:重建论文索引
- **当** 用户点击论文卡片上的"重建"按钮
- **那么** 系统显示确认对话框
- **并且** 用户确认后调用 POST /api/library/papers/{id}/rebuild
- **并且** 成功后论文状态变为 rebuild_pending
- **并且** 显示 toast 提示"已标记重建"

### 需求:重试失败论文
系统必须支持重试处理失败的论文。

#### 场景:重试失败论文
- **当** 用户点击状态为 failed 的论文卡片上的"重试"按钮
- **那么** 系统调用 POST /api/library/papers/{id}/retry
- **并且** 成功后论文状态变为 rebuild_pending
- **并且** 显示 toast 提示"已进入重试队列"
