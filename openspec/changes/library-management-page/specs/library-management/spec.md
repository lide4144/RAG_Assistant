## 新增需求

### 需求:论文列表展示
系统必须提供论文列表展示功能，使用 Card 视图显示论文基本信息。

#### 场景:展示论文卡片
- **当** 用户访问 /library 页面
- **那么** 系统以 Card 形式展示论文列表
- **并且** 每张卡片显示论文标题、状态、专题、导入时间

#### 场景:支持分页浏览
- **当** 论文数量超过每页显示数量
- **那么** 系统显示分页控件
- **并且** 用户可以切换页面浏览

### 需求:论文搜索功能
系统必须支持按标题或来源搜索论文。

#### 场景:通过关键词搜索
- **当** 用户在搜索框输入关键词
- **那么** 系统过滤显示标题或来源包含关键词的论文
- **并且** 搜索结果实时更新（输入后 300ms 防抖）

### 需求:状态筛选功能
系统必须支持按论文状态进行筛选。

#### 场景:按状态筛选论文
- **当** 用户点击状态筛选下拉框
- **那么** 系统显示可选状态列表（ready, parsed, failed, rebuild_pending, deleted）
- **并且** 用户选择状态后只显示该状态的论文

### 需求:专题过滤功能
系统必须支持按专题进行过滤。

#### 场景:按专题过滤论文
- **当** 用户点击专题筛选下拉框
- **那么** 系统显示所有可用的专题列表
- **并且** 用户选择专题后只显示属于该专题的论文

### 需求:实时数据刷新
系统必须支持定时自动刷新论文列表数据。

#### 场景:自动轮询刷新
- **当** 用户停留在 Library 页面
- **那么** 系统每 10 秒自动刷新一次论文列表
- **并且** 页面隐藏时自动暂停轮询以节省资源

### 需求:统计信息展示
系统必须在页面展示知识库统计信息。

#### 场景:显示统计概览
- **当** 用户访问 Library 页面
- **那么** 系统显示论文总数、就绪数量、失败数量、处理中数量
- **并且** 显示各专题的论文分布数量
- **并且** 显示向量后端状态（Qdrant/File）
