## 新增需求

### 需求:多选论文功能
系统必须支持批量选择多篇论文。

#### 场景:单选论文
- **当** 用户点击论文卡片上的复选框
- **那么** 该论文被选中
- **并且** 底部显示批量操作工具栏

#### 场景:全选当前页
- **当** 用户点击"全选本页"按钮
- **那么** 当前页所有论文被选中
- **并且** 显示已选择数量

#### 场景:取消选择
- **当** 用户再次点击已选论文的复选框
- **那么** 该论文取消选中
- **并且** 如果没有选中项，隐藏批量操作工具栏

#### 场景:清空选择
- **当** 用户点击"清空选择"按钮
- **那么** 所有已选论文取消选中
- **并且** 隐藏批量操作工具栏

### 需求:批量删除功能
系统必须支持批量删除多篇论文。

#### 场景:批量删除确认
- **当** 用户选中多篇论文后点击"批量删除"按钮
- **那么** 系统显示确认对话框
- **并且** 对话框列出将被删除的论文标题（最多显示 5 个，其余显示"等 N 篇"）
- **并且** 警告删除将移除元数据、向量索引、专题关联、处理产物

#### 场景:执行批量删除
- **当** 用户确认批量删除
- **那么** 系统调用 POST /api/library/papers/bulk-delete
- **并且** 请求体包含所有选中论文的 ID 列表
- **并且** 显示处理进度
- **并且** 完成后显示结果：成功数量、失败数量
- **并且** 刷新论文列表

### 需求:批量重建功能
系统必须支持批量重建多篇论文的索引。

#### 场景:批量重建确认
- **当** 用户选中多篇论文后点击"批量重建"按钮
- **那么** 系统显示确认对话框
- **并且** 对话框说明将重建选中论文的向量索引

#### 场景:执行批量重建
- **当** 用户确认批量重建
- **那么** 系统调用 POST /api/library/papers/execute-rebuild
- **并且** 请求体包含所有选中论文的 ID 列表
- **并且** 成功后所有选中论文状态变为 rebuild_pending
- **并且** 显示 toast 提示"N 篇论文已标记重建"
- **并且** 清空选择并刷新列表

### 需求:后端批量删除API
系统后端必须提供批量删除论文的 API 端点。

#### 场景:调用批量删除API
- **当** 前端调用 POST /api/library/papers/bulk-delete
- **并且** 请求体包含 paper_ids 数组
- **那么** 后端逐个执行删除操作
- **并且** 返回每个论文的删除结果（成功/失败及错误信息）
- **并且** 返回总体统计：total, succeeded, failed

#### 场景:批量删除部分失败
- **当** 批量删除请求中部分论文删除失败
- **那么** 后端继续处理其他论文
- **并且** 返回详细的失败原因
- **并且** HTTP 状态码为 200，但响应中 ok 字段为 false
