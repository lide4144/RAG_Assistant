## 为什么

当前系统缺少专门的知识库管理界面。用户导入论文后，无法在页面上查看、搜索、删除或重建已导入的论文。Pipeline 页面专注于处理流程，不适合承载论文管理功能。随着论文数量增长，用户急需一个集中管理知识库内容的界面。

## 变更内容

1. **新增后端 API**：创建 `POST /api/library/papers/bulk-delete` 端点，支持批量删除论文
2. **新增前端页面**：创建 `/library` 路由的知识库管理页面
3. **新增导航入口**：在 AppShell 导航栏添加"知识库"选项
4. **新增前端组件**：
   - `PaperCard` - 论文卡片组件
   - `PaperList` - 论文列表容器
   - `PaperDetailModal` - 论文详情弹窗
   - `PaperBulkToolbar` - 批量操作工具栏
   - `PaperFilters` - 筛选组件
5. **新增工具函数**：`usePapersPoll` Hook 用于轮询刷新论文列表

## 功能 (Capabilities)

### 新增功能
- `library-management`: 知识库管理功能，包括论文列表展示、搜索、状态筛选、专题过滤、分页浏览
- `paper-actions`: 单篇论文操作，包括查看详情、删除、重建索引、重试失败任务
- `bulk-operations`: 批量操作功能，支持多选论文进行批量删除和批量重建

### 修改功能
- 无现有功能需求变更

## 影响

- **后端 API**: `app/kernel_api.py` 新增批量删除端点
- **前端路由**: 新增 `/library` 页面
- **前端组件**: 新增 5+ 个组件文件
- **导航**: `AppShell` 导航栏需要添加新入口
- **依赖**: 无破坏性变更，不引入新依赖
- **数据**: 操作的是现有 SQLite 数据库，无需数据迁移
