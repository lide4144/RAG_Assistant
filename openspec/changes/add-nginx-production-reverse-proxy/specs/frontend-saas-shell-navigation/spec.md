## 修改需求

### 需求:系统必须提供全局 SaaS 导航壳层
系统在壳层加载运行态摘要时，必须将管理接口请求稳定路由到 Kernel 服务；当地址未配置或上游返回非 JSON 错误页时，系统必须降级为可读状态提示，禁止抛出前端 JSON 解析异常。在生产部署中，系统必须优先通过同域 `/api/admin/*` 入口访问运行态接口，并由反向代理将请求转发到 Kernel；禁止要求浏览器直接访问 `8000` 端口作为默认生产路径。

#### 场景:未显式配置 Kernel 地址时仍可读取运行态
- **当** 前端未设置 `NEXT_PUBLIC_KERNEL_BASE_URL`
- **那么** 系统必须通过受控默认策略（启动注入或 rewrite）请求到 Kernel 的 `/api/admin/runtime-overview`

#### 场景:生产环境通过同域入口访问运行态
- **当** 系统部署在带有 Nginx 反向代理的生产域名下且未显式配置浏览器侧 Kernel 绝对地址
- **那么** 浏览器必须通过当前站点下的 `/api/admin/runtime-overview` 访问运行态接口，而不是跨域请求单独的 Kernel 域名或端口

#### 场景:运行态接口返回 HTML 错误页
- **当** 运行态接口响应 `content-type` 非 `application/json`
- **那么** 系统必须显示“运行态概览加载失败”类可读提示且不出现 `Unexpected token '<'` 解析错误
