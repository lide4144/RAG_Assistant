## 新增需求

### 需求:系统必须提供单域 Nginx 生产反向代理方案
项目必须提供一个面向生产环境的 Nginx 反向代理方案，使浏览器能够通过单一 HTTPS 域名访问前端页面、管理接口、知识库接口、任务接口和 Gateway WebSocket；禁止要求浏览器在生产环境中直接访问 `8000` 或 `8080` 端口。

#### 场景:用户通过统一域名访问生产站点
- **当** 用户访问生产域名首页或任一工作台页面
- **那么** Nginx 必须将页面请求代理到前端服务，并使浏览器继续通过同一域名访问后续接口

#### 场景:浏览器访问后端接口
- **当** 浏览器请求 `/api/admin/runtime-overview`、`/api/library/import-latest` 或 `/api/tasks/:id/cancel`
- **那么** Nginx 必须将这些请求转发到 Kernel 服务并保持响应对浏览器表现为同域请求

### 需求:Nginx 必须支持 WebSocket 代理到 Gateway
项目提供的 Nginx 方案必须将 `/ws` 路径代理到 Gateway，并且必须包含 WebSocket 升级所需的 `Upgrade`、`Connection` 和 `Host` 头转发配置；禁止仅提供 HTTP 代理示例而省略 WebSocket 透传要求。

#### 场景:浏览器建立聊天或流水线 WebSocket
- **当** 浏览器通过 `wss://<生产域名>/ws` 建立连接
- **那么** Nginx 必须将该连接升级并转发到 Gateway，使聊天和流水线任务事件能够正常传输

### 需求:生产反代文档必须包含部署与验证步骤
项目必须提供与 Nginx 反向代理对应的部署文档，文档必须包含内部服务监听约定、Nginx 配置示例、环境变量说明以及最小验证步骤。

#### 场景:运维首次部署生产环境
- **当** 运维按照文档部署该项目
- **那么** 文档必须让其能够完成 Nginx 配置、启动内部服务，并通过页面访问与接口检查确认部署可用
