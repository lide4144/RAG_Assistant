## 新增需求

### 需求:系统必须提供 Pipeline Workbench 一级入口
系统必须在产品化工作台中提供 `Pipeline` 一级入口，用于承载导入、处理与建图任务，禁止仅通过聊天页面承载全部任务操作。

#### 场景:用户可从工作台进入处理流程
- **当** 用户进入研究助手工作台
- **那么** 界面必须可在一级导航中访问 Pipeline 区域并查看当前任务状态

### 需求:系统必须在工作台展示长任务实时状态
系统必须在 Pipeline 区域持续展示长任务的阶段状态、进度与最近更新时间，禁止仅显示无限等待态。

#### 场景:图构建运行中可见状态
- **当** 图构建任务处于 `running`
- **那么** Pipeline 区域必须显示当前阶段、进度值与累计耗时

## 修改需求

### 需求:长耗时任务必须提供可观察进度
系统必须在导入、解析、索引等长耗时本地任务中提供实时进度反馈，且必须支持在统一任务中心按任务查看阶段状态、失败原因与重试入口，禁止仅显示无限期等待状态。

#### 场景:导入流程显示阶段进度
- **当** 用户发起论文导入并进入多阶段处理
- **那么** 界面必须持续显示阶段进度与当前处理状态

#### 场景:失败任务可在任务中心恢复
- **当** 任一长任务执行失败
- **那么** 任务中心必须显示失败阶段与错误摘要，并提供可执行的重试入口
