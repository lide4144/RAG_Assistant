## 新增需求

### 需求:系统必须在导入完成后显示后续处理链路状态
系统必须在导入结果中展示后续处理链路（切块清洗、图构建）的当前状态与可执行动作，禁止将用户停留在“导入完成”但无下一步引导的状态。

#### 场景:导入成功后显示下一步动作
- **当** 用户完成论文导入
- **那么** 系统必须展示“继续处理/开始图构建/进入聊天验证”等可执行下一步

## 修改需求

### 需求:系统必须提供低门槛论文导入入口
系统必须在 Library 中提供统一导入入口，支持本地文件导入，并以任务语义反馈处理结果；当导入触发后续流水线任务时，系统必须展示该任务的状态与进度摘要。

#### 场景:导入后给出可理解结果
- **当** 用户完成论文导入
- **那么** 系统必须反馈成功/失败数量与可继续动作（例如立即提问）

#### 场景:导入结果分类可见
- **当** 一次导入包含新增、重复与失败样本
- **那么** 系统必须分别展示新增数量、已存在跳过数量、失败数量及原因

#### 场景:导入后任务摘要可见
- **当** 导入触发切块清洗或图构建
- **那么** 系统必须展示后续任务当前状态（queued/running/succeeded/failed）与最近更新时间
