## 新增需求

### 需求:入库主产物写入必须原子落盘
系统必须对 `chunks.jsonl`、`papers.json` 等主产物执行“临时文件 + 原子替换”写入协议，禁止在目标文件上直接追加或覆盖。

#### 场景:主产物写入成功
- **当** 入库流程完成主产物生成
- **那么** 目标文件必须为完整一致版本且不存在半成品可见窗口

#### 场景:主产物写入失败
- **当** 写入中途发生异常
- **那么** 系统必须保留旧版本目标文件并清理当前轮次临时文件

### 需求:入库写入与索引更新必须具备本地互斥
系统必须在本地写入与索引更新入口使用文件锁保护同一目标资源，禁止并发实例破坏数据一致性。

#### 场景:重复点击触发并发导入
- **当** 用户在短时间内重复触发导入并命中同一输出目录
- **那么** 系统必须阻止并发写入并向后触发方返回明确冲突提示

### 需求:Chunk 校验必须包含类型与约束
系统必须在 `chunk_id`、`paper_id`、`page_start`、`text` 字段存在校验之外执行类型与语义约束校验。

#### 场景:字段类型不合法
- **当** `page_start` 为非整数或 `chunk_id` 为 `null`
- **那么** 系统必须将该记录判定为非法并输出可定位的行级错误

#### 场景:文本为空或仅空白
- **当** `text` 为空字符串或仅包含空白字符
- **那么** 系统必须将该记录判定为非法并拒绝通过校验

### 需求:校验流程必须支持 fail-fast 阈值
系统必须支持 `max_errors` 阈值，在错误密集输入下提前终止并输出错误摘要。

#### 场景:超阈值停止校验
- **当** 校验错误数达到配置阈值
- **那么** 系统必须停止继续扫描并返回已收集错误及“已提前停止”标记
