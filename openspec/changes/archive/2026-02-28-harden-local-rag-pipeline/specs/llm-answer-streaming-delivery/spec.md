## 新增需求

### 需求:前端必须消费并展示回答增量流
当回答流式模式开启时，系统必须在 UI 侧持续渲染增量内容，禁止等待完整回答后一次性展示。

#### 场景:UI 实时展示流式内容
- **当** 回答进入流式生成路径
- **那么** 用户必须在最终完成前看到持续更新的文本输出

### 需求:流式链路中断必须降级并保持上下文连续
系统必须在流式连接中断、超时或空流时返回可恢复结果并保留当前会话上下文，禁止清空或打断对话链。

#### 场景:流式中断后恢复交互
- **当** 回答流式连接在完成前异常中断
- **那么** 系统必须输出降级提示与可继续提问入口，且历史消息保持可用
