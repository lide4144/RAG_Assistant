## 修改需求

### 需求:系统必须在检索前执行控制意图路由
系统必须将控制意图识别升级为语义路由，并输出结构化结果（至少包含 `intent_type` 与必要参数）；禁止仅依赖关键词/正则列表作为唯一识别手段。

#### 场景:表达变体仍可识别控制意图
- **当** 用户使用非预设措辞表达格式控制意图（如“换成表格展示”）
- **那么** 系统必须识别为控制意图并输出结构化参数

### 需求:系统必须提供可配置的路由开关与回退
系统必须支持语义路由开关，并在语义路由失败时回退到最小可用路径，不得中断主流程。

#### 场景:语义路由失败回退
- **当** 路由模型调用失败或输出不合法
- **那么** 系统必须回退到默认 QA 路径并记录路由失败观测信息

### 需求:系统必须提供意图路由可观测字段
系统必须输出意图路由观测字段，至少包含路由来源（semantic/model/rule_fallback）、命中意图、置信度与回退标识。

#### 场景:路由观测字段完整
- **当** 一次请求完成意图判定
- **那么** 系统输出必须包含可审计的路由观测字段

## 移除需求

### 需求:关键词列表主导的控制意图路由
**Reason**: 关键词列表难以覆盖自然语言变体，维护成本高且漏判明显。
**Migration**: 主路径切换到语义路由，关键词规则保留为紧急降级回退。
