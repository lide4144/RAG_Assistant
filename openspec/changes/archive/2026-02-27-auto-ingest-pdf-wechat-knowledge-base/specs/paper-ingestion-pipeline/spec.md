## 修改需求

### 需求:批量论文摄取命令
系统必须提供统一摄取命令用于批量处理 PDF 与 URL 两类输入。命令必须支持目录内 PDF 发现，同时支持通过参数或清单文件传入 URL 列表；两类输入必须进入同一入库编排并输出统一产物。

#### 场景:混合输入统一入库
- **当** 用户同时提供 PDF 输入目录与 URL 列表
- **那么** 系统必须在一次运行中处理两类输入并生成统一的文档映射与分块产物

#### 场景:仅 URL 输入可运行
- **当** 用户未提供 PDF 目录但提供有效 URL 列表
- **那么** 系统必须完成 URL 入库流程并输出可用于后续检索的产物

### 需求:输入文件发现与过滤
系统必须处理输入中的 `.pdf` 文件与有效 URL，且必须忽略不支持的文件类型与非法 URL 条目，不得中断整批处理。

#### 场景:混合输入包含无效条目
- **当** 输入集合包含 `a.pdf`、`b.txt` 与非法 URL
- **那么** 系统必须处理 `a.pdf`、跳过无效条目并记录跳过原因

### 需求:论文元数据映射输出
系统必须为每篇已处理文档写入统一映射记录，并在原有 `paper_id`、`title`、`path` 基础上扩展 `source_type` 与 `source_uri`（或等价字段）以区分 PDF 与 URL 来源。

#### 场景:URL 文档映射完整落盘
- **当** 一条 URL 文档处理成功
- **那么** 文档映射记录必须包含来源类型与来源 URI 字段并可查询
