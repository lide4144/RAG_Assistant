## 新增需求

### 需求:系统必须支持通过 URL 导入网页文档
系统必须支持用户提交网页 URL 进行导入，首版必须覆盖微信公众号文章链接的主流程可用性。

#### 场景:提交有效公众号链接
- **当** 用户提交可访问的公众号文章 URL
- **那么** 系统必须完成抓取与正文抽取，并纳入统一文档入库流程

### 需求:系统必须输出可解释的链接导入失败原因
当 URL 导入失败时，系统必须返回结构化失败原因（例如网络失败、访问受限、解析失败、正文为空），禁止仅返回不可解释的通用失败提示。

#### 场景:链接抓取失败
- **当** 目标 URL 无法访问或正文抽取失败
- **那么** 系统必须输出可解释失败原因并提供可继续动作（重试或忽略）

### 需求:系统必须保留 URL 来源可追溯元数据
系统必须在入库记录中保存来源 URL 及必要的抓取元数据（如抓取时间、标题、来源类型），以支持后续审查与回溯。

#### 场景:URL 文档入库后可追溯
- **当** URL 文档入库成功
- **那么** 系统必须能从文档记录查询到原始来源 URL 与抓取元数据
