## 为什么

当前系统的入库路径主要围绕本地论文与工程化流程，无法直接承接用户最常见的资料输入方式（论文 PDF + 公众号链接）。同时，检索粒度主要依赖 chunk，缺少论文级摘要索引，导致“先快速定位相关论文，再下钻证据”的效率不足。

## 变更内容

- 扩展入库入口：支持 `PDF 文件` 与 `网页链接（首版重点支持微信公众号文章链接）` 的统一导入。
- 为每篇文档新增独立的论文级摘要产物（paper summary artifacts），用于快速检索、主题导航与结果聚合。
- 将检索流程调整为“双层检索”：先在论文级摘要层粗召回，再在命中论文内进行 chunk 级证据检索与重排。
- 保持证据约束：摘要层用于召回与组织，回答中的关键结论仍必须绑定 chunk 级可追溯 citation。
- 现代化多轮会话状态：引入 Redis 会话存储、分层记忆（recent/summary/semantic）与轻量状态机，替代本地文件 + 纯滑窗上下文。
- 将查询重写升级为“独立问题重写（Standalone Query Rewrite）”，由小模型将指代问题改写为可检索的完整问题。
- 将意图路由升级为语义路由与结构化输出，减少关键词硬编码规则对真实表达的误判。
- 调整用户导入反馈语义，确保普通用户可理解导入成功/失败与下一步操作。

## 功能 (Capabilities)

### 新增功能
- `paper-summary-artifacts`: 为每篇入库文档持久化论文级摘要对象（标题、摘要、要点、关键词、来源、版本与新鲜度标识），并支持检索读取。
- `web-link-ingestion`: 支持通过 URL 导入网页文章（首版重点微信公众号链接），完成抓取、正文抽取、元数据记录与可追溯入库。

### 修改功能
- `paper-ingestion-pipeline`: 从仅 PDF 入库扩展为 PDF + URL 双输入通道，并统一写入文档元数据与后续分块处理路径。
- `rag-baseline-retrieval`: 增加“summary 粗召回 -> chunk 精检索”的两阶段检索策略。
- `output-consistency-evidence-allocation`: 明确摘要层不能直接充当最终事实引用，回答引用必须来自 `evidence_grouped` 的 chunk 证据。
- `multi-turn-session-state`: 从本地文件与固定滑窗升级为 Redis 会话存储 + 分层记忆 + 显式状态机管理。
- `query-rewriting`: 从规则拼接升级为 LLM 独立问题重写，提升多轮指代与中英混杂场景下检索召回。
- `control-intent-routing`: 从关键词规则升级为语义路由与结构化意图输出。

## 影响

- 受影响代码：`app/ingest*`、检索与排序链路、会话状态管理、查询重写与意图路由模块、UI 导入入口与导入反馈文案、结果组织与引用映射逻辑。
- 受影响数据：新增论文摘要存储文件/索引、URL 来源元数据字段，现有 `papers/chunks` 产物需要兼容扩展字段。
- 受影响验证：需新增链接抓取可用性与降级测试、摘要索引命中测试、双层检索回归测试、会话并发与状态恢复测试、重写质量评测与语义路由准确率测试、引用合规测试。
