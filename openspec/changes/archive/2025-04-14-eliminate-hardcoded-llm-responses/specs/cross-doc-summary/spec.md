## 新增需求

### 需求:跨文档总结必须通过LLM生成
系统禁止在跨文档总结场景下直接拼接证据片段输出。所有跨文档总结必须通过LLM生成自然语言回答。

#### 场景:多论文贡献对比
- **当** 用户请求"对比这些论文的方法差异"
- **那么** 系统必须调用LLM生成对比分析
- **并且** 禁止直接输出格式化的证据列表

#### 场景:实验结果总结
- **当** 用户请求"总结这些论文的实验结论"
- **那么** 系统必须调用LLM生成实验结论的总结
- **并且** LLM的回答必须自然流畅，体现各论文之间的关系

## 修改需求

### 需求:禁止证据直接拼接输出
跨文档总结功能必须移除直接拼接证据片段作为回答输出的逻辑。证据只能作为上下文输入给LLM。

#### 场景:生成对比表
- **当** 系统检测到用户请求生成对比表
- **那么** 禁止调用`_build_assistant_summary_answer`直接拼接证据
- **并且** 必须构建LLM提示词，指示LLM生成表格格式的对比

#### 场景:生成主题列表
- **当** 系统需要展示多个论文的主题
- **那么** 禁止直接输出"基于当前可追溯证据，优先可关注这些主题："
- **并且** 必须由LLM生成主题介绍和分析

### 需求:证据上下文格式化
跨文档总结必须将多文档的证据整合为结构化的上下文，供LLM理解和参考。

#### 场景:多源证据整合
- **当** 从多个论文检索到相关证据
- **那么** 按论文分组组织证据
- **并且** 为每组证据标注来源和引用编号

#### 场景:证据相关性排序
- **当** 整合多源证据
- **那么** 按相关性分数对证据进行排序
- **并且** 在上下文中优先呈现高相关性证据

## 移除需求

### 需求:机械化主题列表生成
**Reason**: 原逻辑直接拼接证据生成"基于当前可追溯证据，优先可关注这些主题"等机械化回复
**Migration**: 使用LLM基于证据上下文生成自然的主题介绍和分析

### 需求:硬编码追问建议
**Reason**: 原逻辑硬编码了三条追问建议（"这些方向中哪个与你当前任务最相关？"等）
**Migration**: 由LLM根据证据内容动态生成个性化的追问建议
