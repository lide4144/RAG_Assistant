## 上下文

当前问答链路对“自然口语+开放总结”场景过于保守，常在 `sufficiency-gate` 前后触发拒答或连续澄清，导致用户必须手工收窄问题模板才能获得输出。最近的真实交互验证显示，该行为在首轮和多轮中都存在，并且会受到上一轮澄清信号污染。

本变更涉及 `rewrite`、`intent routing`、`sufficiency gate`、`session state`、`UI trace` 多模块联动，目标是在不破坏可追溯性的前提下，让默认体验更像论文助理。

## 目标 / 非目标

**目标：**
- 让自然提问默认进入“助理优先”路径：先给可追溯主题地图/方向建议，再在必要时澄清。
- 将“证据不足”时的默认动作从直接拒答改为“最小澄清优先”，并限制澄清问题数量。
- 降低上一轮控制意图、澄清约束对下一轮自然提问的污染，保证多轮可用性。
- 保持输出可审查：决策、证据来源、澄清触发原因仍需在 trace/UI 可见。

**非目标：**
- 不引入新的外部模型或新供应商依赖。
- 不改变底层索引格式与离线数据管线。
- 不在本变更中追求“任何问题都必须回答”；仍保留高风险场景拒答能力。

## 决策

### 决策 1: 引入“助理优先”回答策略层
- 选择：在决策层增加 `assistant_summary` 路径，对“宽口径总结类问题”允许基于高相关证据先返回可追溯摘要和后续提问建议。
- 备选：
  - A. 继续完全依赖现有 `answer/refuse/clarify`，不新增中间策略。
  - B. 直接放宽拒答阈值，减少 `refuse`。
- 取舍：A 无法改善体验；B 易导致幻觉。`assistant_summary` 可在可追溯约束下改善可用性。

### 决策 2: 将 Gate 的缺口处理改为“最小澄清优先”
- 选择：当缺口可通过一次澄清补齐时，优先输出 1 个澄清问题，而非直接拒答。
- 备选：
  - A. 维持当前 `refuse` 优先。
  - B. 允许多问题澄清（2+）一次抛出。
- 取舍：A 用户成本高；B 交互负担大。单问题澄清更贴近日常使用。

### 决策 3: 增加会话“意图去污染”规则
- 选择：当当前轮检测为自然总结意图时，重写阶段必须降权上一轮 `clarify/numeric_constraint` 信号，仅保留主题实体锚点。
- 备选：
  - A. 完全继承上一轮约束。
  - B. 完全丢弃历史。
- 取舍：A 容易误导当前轮；B 丢失上下文。降权继承平衡可用性与连续性。

### 决策 4: 通过可观测字段量化体验改进
- 选择：新增/强化 `assistant_mode_used`、`clarify_count`、`gate_trigger_reason`、`history_constraint_dropped` 等追踪字段。
- 备选：仅凭人工 UI 观察判断改进效果。
- 取舍：仅人工观察不可回归，必须有可比指标。

### 决策 5: 设置“连续澄清终止”与“强制可用输出”策略
- 选择：在同一会话主题下设置连续澄清上限（默认 2 轮）；达到上限后禁止继续 `clarify`，必须输出低置信但可追溯的主题化回答（含不确定性标记与引用）。
- 备选：
  - A. 不设上限，持续澄清直到用户给足约束。
  - B. 达到上限后直接拒答。
- 取舍：A 会造成日常使用卡死；B 体验仍是阻断式。强制可用输出更符合“论文助理”目标。

## 风险 / 权衡

- [风险] 助理优先策略可能放宽过度，出现泛化性总结偏离证据。  
  → 缓解措施：要求主题摘要必须绑定 `answer_citations`；无可绑定证据时退回最小澄清。

- [风险] 去污染规则误删有效上下文，降低多轮连续性。  
  → 缓解措施：仅降权“约束型信号”，保留实体锚点与最近主题轮次。

- [风险] 决策分支增加后可维护性下降。  
  → 缓解措施：在 trace 中输出分支命中原因，并补充针对性回归样例。

## Migration Plan

1. 先落地规格与任务，确保行为边界清晰。  
2. 按“路由/重写 → gate → 会话状态 → UI 可观测”顺序实现。  
3. 在回归集中新增“自然首问总结”“多轮追问建议”“最小澄清”场景并与现有拒答基线对比。  
4. 若出现误答回归，回滚到旧 gate 阈值与旧 routing 分支，并保留新增 trace 字段用于复盘。

## 结果导向验收门槛

- 自然首问（“总结方向 + 下一步怎么问”）不得直接返回模板化拒答文案，必须返回：
  - 可追溯主题化回答，或
  - 单问题最小澄清（仅 1 条）。
- 两轮口语追问场景中，至少 1 轮必须产出可追溯主题化回答，禁止连续两轮模板化拒答。
- `assistant_mode_used=true` 时，默认不得落入直接 `refuse`，除非存在明确不可澄清的语料外风险信号。
- 可观测字段（`assistant_mode_used`、`gate_trigger_reason`、`history_constraint_dropped`）必须与最终决策一致，不得出现“字段命中但行为未改变”的伪通过。
- 同一主题会话中，连续 `clarify` 不得超过 2 轮；达到上限后下一轮必须输出低置信可用回答（含引用与不确定性说明），不得继续追问澄清。
