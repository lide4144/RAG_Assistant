## 新增需求

### 需求:系统必须在多轮中抑制约束污染
系统必须在会话状态中区分“主题锚点”与“临时澄清约束”。当当前轮为自然总结/开放式提问时，系统必须降权或剔除上一轮临时澄清约束（如 numeric、subject_constraint），并保留主题锚点用于连续性。

#### 场景:自然提问触发约束降权
- **当** 上一轮为 `clarify` 且当前轮是开放式总结提问
- **那么** 系统必须保留主题实体并降权上一轮临时约束，不得强制沿用上一轮澄清模板

#### 场景:约束污染处理可观测
- **当** 系统执行了约束降权或剔除
- **那么** run trace 必须输出对应可序列化信号字段，供 UI 审查

### 需求:系统必须维护主题级澄清计数状态
系统必须在会话状态中维护主题级连续澄清计数，并向 Gate 暴露该计数与是否命中上限。主题切换后计数必须重置，避免旧主题澄清状态污染新主题问答。

#### 场景:同主题连续澄清计数递增
- **当** 当前主题下连续发生 `decision=clarify`
- **那么** 会话状态中的 `clarify_count` 必须逐轮递增并可被后续决策读取

#### 场景:主题切换触发计数重置
- **当** 系统判定用户问题主题发生切换
- **那么** 必须将旧主题的连续澄清计数清零或切换到新主题计数上下文

## 修改需求

### 需求:滑动窗口历史供重写
系统必须在 rewrite 阶段读取最近 N 轮（默认 N=3）脱水历史，并生成 `standalone_query`。若当前问题含指代，`standalone_query` 必须补齐关键实体，且禁止拼接历史回答整段文本。若上一轮 `decision=clarify`，系统必须先执行 clarify 合并逻辑，再进行元问题护栏判断；但当当前轮识别为开放式总结提问时，系统必须对上一轮澄清约束执行降权处理。

#### 场景:指代追问补齐实体
- **当** 当前输入为代词追问且历史包含实体
- **那么** `standalone_query` 必须包含该实体并保持当前问题意图

#### 场景:开放式总结时不继承澄清硬约束
- **当** 上一轮为 `clarify` 且当前轮输入为“总结方向/下一步问什么”等开放式提问
- **那么** 重写结果不得强制继承上一轮 numeric 或主体限定约束

### 需求:重写输入必须包含上轮决策与告警信号
系统必须在 rewrite 输入中提供 `last_turn_decision` 与 `last_turn_warnings`，用于状态感知转写与补证据优先级判定。系统必须额外提供“是否应用约束去污染”的状态字段，禁止静默处理。

#### 场景:状态信号传递到 rewrite
- **当** 会话存在上一轮记录
- **那么** 系统必须向 rewrite 阶段传入上一轮 `decision` 与 `output_warnings` 的脱水映射

#### 场景:去污染状态可审查
- **当** 系统根据当前轮意图丢弃了上一轮临时约束
- **那么** trace 中必须包含 `history_constraint_dropped=true` 或等价字段
