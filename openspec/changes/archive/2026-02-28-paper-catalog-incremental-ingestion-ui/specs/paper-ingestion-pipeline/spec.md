## 新增需求

### 需求:入库流程必须支持增量合并
系统必须在导入时读取现有 `papers/chunks` 产物并进行去重合并，禁止默认覆盖历史已收录语料。

#### 场景:重复导入同一论文
- **当** 用户再次导入已存在论文
- **那么** 系统必须保留历史语料并将该论文标记为“已存在跳过”

#### 场景:新论文导入
- **当** 用户导入未收录论文
- **那么** 系统必须将该论文及其 chunks 追加到现有语料集合

### 需求:系统必须使用稳定论文标识
系统必须为同一来源内容生成稳定 `paper_id`，不得依赖临时目录路径导致标识漂移。

#### 场景:相同文件不同临时路径导入
- **当** 同一 PDF 经由不同临时路径重复导入
- **那么** 系统必须识别为同一论文并复用相同稳定 `paper_id`

## 修改需求

### 需求:论文元数据映射输出
系统必须为每篇已处理文档写入统一映射记录，并在原有 `paper_id`、`title`、`path` 基础上扩展 `source_type` 与 `source_uri`（或等价字段）以区分 PDF 与 URL 来源。

#### 场景:URL 文档映射完整落盘
- **当** 一条 URL 文档处理成功
- **那么** 文档映射记录必须包含来源类型与来源 URI 字段并可查询

#### 场景:增量写入保持映射连续性
- **当** 多次导入在不同时间发生
- **那么** `papers.json` 必须保留历史记录并保持可追溯连续性
