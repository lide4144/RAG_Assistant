## 为什么

当前 Library 仅支持“导入+专题分组”的轻量交互，缺少完整的论文目录管理能力（搜索、筛选、状态、元数据可视化、重复识别）。同时，Web 端导入实际为覆盖式重建，导致已收录语料被替换、同一论文重复导入识别不稳定，用户预期与系统行为不一致。

现在需要把“论文管理”从辅助功能升级为一等能力，并将导入流程调整为增量更新，以支持长期维护个人论文知识库。

## 变更内容

- 新增论文目录管理能力：提供“已收录论文”页面化管理（列表、检索、筛选、详情、专题批量操作）。
- 调整 Web 导入语义为增量导入：新导入论文追加到现有语料，默认不覆盖既有论文。
- 引入稳定论文标识策略，避免临时路径变化导致同一论文生成不同 `paper_id`。
- 明确导入去重与冲突策略（同文件重复、同内容重复、同名不同内容）。
- 将索引更新策略分层：先支持“数据增量 + 索引重建”的稳定版本，后续保留“真增量索引”扩展位。

## 功能 (Capabilities)

### 新增功能
- `paper-catalog-management`: 论文目录管理页与目录服务，支持检索、筛选、详情查看、专题关联与可观测状态。

### 修改功能
- `library-quick-ingestion`: 导入流程从“覆盖式”升级为“增量式”，并补充重复识别、导入结果分类反馈。
- `paper-ingestion-pipeline`: 入库主数据从一次性重写调整为合并写入与稳定标识策略，保证长期数据连续性。

## 影响

- 受影响代码：`app/ui.py`、`app/library.py`、`app/ingest.py`、`app/parser.py`、部分测试。
- 受影响数据：`data/processed/papers.json`、`data/processed/chunks*.jsonl`、`data/library_topics.json`、`data/indexes/*`。
- 兼容性影响：需要定义旧 `paper_id` 与新稳定标识并存或迁移策略，避免专题映射与历史会话失联。
