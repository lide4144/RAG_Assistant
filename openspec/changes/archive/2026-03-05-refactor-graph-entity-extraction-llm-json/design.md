## 上下文

当前图构建流程在 `app/graph_build.py` 通过正则和规则从 `clean_text` 抽取实体，仅覆盖大写缩写、连字符数字词与 CamelCase。该方式在专业术语、多语言缩写、上下文判别（同形异义）方面准确率不足，直接影响实体共现边质量。与此同时，图构建是批处理路径，若直接并发调用外部 LLM API，容易触发 429/超时并导致构建失败。

因此本变更需要在“缺失 `entities` 字段”的分支引入结构化 LLM 抽取，并通过统一 schema 校验、异常处理和并发限制保证稳定性。

## 目标 / 非目标

**目标：**
- 在缺失 `entities` 时，使用 LLM 输出严格 JSON，并由 Pydantic 校验为统一实体结构（`entity_name`, `entity_type`）。
- 提供异步批量抽取能力，支持受控并发（信号量）和批次聚合（`asyncio.gather`）。
- 对 API 限流、超时、JSON 解析失败等异常提供降级与不中断处理，保证图构建流程可继续。
- 保持现有实体共现建边接口兼容，不改变下游边构建的输入契约（仍消费实体字符串集合或等价结构）。

**非目标：**
- 不在本变更中引入复杂实体消歧、跨 chunk 实体归一化或知识库链接。
- 不改造图边打分公式与 `top_m` 截断策略。
- 不覆盖完整提示词管理平台，仅提供满足当前构图需求的最小提示词与输出约束。

## 决策

1. **采用 Pydantic 作为 LLM 输出契约层**
   - 决策：定义 `EntityExtractionItem(entity_name: str, entity_type: str)` 与容器模型（如 `EntityExtractionResult`）用于解析与校验。
   - 理由：Pydantic 提供强校验与可读错误，能将 LLM 输出不确定性收敛到可处理异常。
   - 备选方案：
     - 仅手写 `json.loads` + dict 校验：实现快但错误分支分散、可维护性差。
     - 使用 dataclass：缺少原生验证与详细错误上下文。

2. **采用 OpenAI 兼容 Chat Completions + JSON 输出约束**
   - 决策：通过 OpenAI 兼容接口发起请求，使用 `response_format={"type":"json_object"}`（或等效 JSON schema 约束能力）强制模型返回 JSON。
   - 理由：复用现有生态，便于切换兼容提供商并统一调用方式。
   - 备选方案：
     - 继续规则抽取：准确性上限低。
     - 使用函数调用工具模式：更强约束但集成复杂度更高；当前阶段先采用 JSON object 约束。

3. **批处理并发模型采用 Semaphore + gather**
   - 决策：每个 chunk 抽取任务在协程内执行，外层以 `asyncio.Semaphore(max_concurrency)` 控制并发，批次通过 `asyncio.gather(..., return_exceptions=True)` 汇总。
   - 理由：易于理解与调优，可在高吞吐与限流保护间平衡。
   - 备选方案：
     - 无并发限制：风险高，易触发限流。
     - 串行调用：稳定但吞吐过低。

4. **异常处理策略采用“记录 + 降级为空实体”**
   - 决策：对于单 chunk 的请求失败（429、超时、非 JSON、schema 校验失败），记录结构化日志并返回空实体列表，不中断全量构图。
   - 理由：图构建属于离线流程，单点失败不应导致全局失败。
   - 备选方案：
     - 任一失败即整体失败：可见性强但可用性差。
     - 无限重试：可能放大限流并延长任务时间。

## 风险 / 权衡

- [外部 API 成本与时延上升] → 通过并发上限、可配置模型与仅在缺失 `entities` 时触发来控制调用规模。
- [模型输出漂移导致类型不一致] → 使用严格 schema 校验与失败降级，定期回放样本评估提示词。
- [限流高峰导致吞吐下降] → 对 429/超时设置有限重试与退避，并允许配置较低并发上限。
- [降级为空实体可能降低图连通性] → 增加失败指标与告警，后续可引入重试任务或回补机制。

## Migration Plan

1. 在 `app/graph_build.py` 新增 Pydantic 模型与异步抽取函数，直接替换缺失 `entities` 时的旧规则路径。
2. 接入 OpenAI 兼容客户端配置（base URL、api key、model、timeout、max_concurrency）。
3. 将“缺失 `entities`”分支切换到 LLM 抽取路径，确保输出可被现有共现建边逻辑消费。
4. 通过小样本回归验证实体覆盖率、构图耗时、失败率，并补充关键异常场景测试（429、非 JSON、schema 错误）。
5. 回滚策略：不再依赖旧规则路径开关；若调用稳定性波动，则下调并发、降低重试并允许失败降级为空实体，待问题修复后恢复参数。
