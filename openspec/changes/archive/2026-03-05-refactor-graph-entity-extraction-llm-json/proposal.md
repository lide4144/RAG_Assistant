## 为什么

当前 `app/graph_build.py` 在图构建时使用正则与规则抽取实体，覆盖面和稳健性有限，难以适配复杂术语与多样文本模式。随着实体共现边对检索质量的影响增大，需要将实体抽取升级为可结构化约束的 LLM 提取方案，并同时保证并发调用下的稳定性与限流安全。

## 变更内容

- 将缺失 `entities` 时的规则/正则抽取逻辑替换为基于 LLM 的结构化抽取流程。
- 引入 Pydantic 实体模型（包含 `entity_name` 与 `entity_type`）作为标准输出契约，并对模型输出进行校验。
- 新增异步实体抽取函数，面向 OpenAI 兼容 API，要求模型以 JSON 输出实体列表。
- 增加异常处理与并发控制能力（例如基于信号量与 `asyncio.gather` 的批处理），降低限流与瞬时失败导致的构图中断风险。

## 功能 (Capabilities)

### 新增功能
- `llm-entity-extraction-for-graph`: 为图构建阶段新增结构化 LLM 实体抽取能力，定义输出 schema、错误降级与并发策略。

### 修改功能
- `chunk-graph-construction`: 将“缺失 entities 时必须规则抽取”的需求修改为“缺失 entities 时必须进行基于 schema 的 LLM 抽取，并具备并发与异常控制”。

## 影响

- 受影响代码：`app/graph_build.py` 及其相关实体抽取调用链。
- 受影响行为：实体共现边的实体来源从规则提取迁移为 LLM 结构化输出。
- 受影响依赖：需要 Pydantic 与 OpenAI 兼容客户端（或现有兼容实现）支持。
- 运行影响：新增外部 API 调用、并发限制与重试/降级策略配置需求。
