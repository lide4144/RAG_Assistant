## 新增需求

### 需求:Query Rewriting 基础产出
系统必须将输入问题 `Q` 改写为检索查询 `Q'`，并同时输出 `keywords/entities` 结构供检索增强使用。

#### 场景:给定问题产生改写结果
- **当** 用户输入任意问题 `Q`
- **那么** 系统必须返回非空 `Q'`，并返回可序列化的 `keywords/entities` 字段

### 需求:术语保留策略
系统必须保留原问题中的英文缩写、公式样式与关键数字指标（如 accuracy/F1/BLEU），禁止在改写中删除这些信息。

#### 场景:缩写与指标保留
- **当** 问题包含 `F1`、`BLEU`、`Top-1`、`AUC` 等术语
- **那么** 改写结果 `Q'` 必须保留这些术语或其等价字面表示

### 需求:关键词扩展策略
系统必须支持基于同义词/近义词/领域词典的关键词扩展，并将扩展结果写入 `keywords/entities`。

#### 场景:同义词扩展
- **当** 问题中出现可扩展术语（如 citation/reference）
- **那么** 系统必须在 `keywords/entities` 中输出扩展词集合用于检索增强

### 需求:问句转检索句策略
系统必须将冗余问句转换为检索导向表达，保留核心语义并删除无效礼貌/冗余成分。

#### 场景:问句去冗余
- **当** 用户问题包含礼貌或填充语（如“请问”“能不能帮我看看”）
- **那么** `Q'` 必须去除冗余并保留核心检索意图

### 需求:可选 LLM 改写开关
系统必须提供可选 LLM 改写开关，默认关闭；当开关关闭时必须仅使用规则改写。

#### 场景:LLM 开关关闭
- **当** 未启用 LLM 改写开关
- **那么** 系统必须只执行规则改写并产出 `Q'`
