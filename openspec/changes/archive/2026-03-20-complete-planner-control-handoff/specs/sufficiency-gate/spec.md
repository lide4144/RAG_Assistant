## MODIFIED Requirements

### 需求:Sufficiency Gate 决策输出
系统必须将 `Sufficiency Gate（充分性门控）` 定义为 `guardrail（护栏）` 组件，而不是最终用户交互裁判。Sufficiency Gate 必须在回答前基于 `question（问题）` 与 `top_n evidence（Top-N 证据）` 产出结构化约束对象，至少包含 `reason（原因）`、`reason_code（原因代码）`、`severity（严重度）`、`clarify_questions（澄清问题，可选）`、`output_warnings（告警）` 与必要的主题/要素覆盖信息。禁止要求其以 `answer / refuse / clarify（回答 / 拒答 / 澄清）` 作为唯一顶层决策口径；最终用户可见姿态必须由 `planner / policy（规划器 / 策略层）` 基于该约束对象统一决定。

#### 场景:输出结构合法
- **当** 系统收到一次问答请求并完成证据聚合
- **那么** Sufficiency Gate 必须返回结构化约束对象及其可审计字段，而不得直接作为最终用户姿态裁判

#### 场景:可选澄清问题数量固定为 1
- **当** Sufficiency Gate 判定当前约束可通过单轮澄清缓解
- **那么** 输出中的 `clarify_questions` 必须恰好为 1 个，供上层 `planner / policy（规划器 / 策略层）` 选择是否采用

#### 场景:发布统计口径使用最终用户姿态
- **当** 系统执行增长评测汇总
- **那么** 统计必须基于最终用户可见姿态口径，而不得将 Sufficiency Gate 的内部约束结果直接当作唯一最终决策

### 需求:主题不匹配必须触发不足
当 `top_n evidence（Top-N 证据）` 与问题主题相关性低于门限时，Sufficiency Gate 必须产出“主题不匹配”的结构化约束，并禁止将其静默视为可直接回答。对于可通过单一澄清补齐主题约束的场景，约束对象必须提供 1 个候选澄清问题；仅当无法形成有效澄清路径时，才允许将该约束标记为高严重度阻断。对于控制意图轮次，Sufficiency Gate 必须使用锚定主题查询（`anchor_query（锚定查询）` 或等价字段）作为主题匹配输入来源，而非直接使用控制语句本身。

#### 场景:相关性低但可澄清时提供候选澄清
- **当** 证据主题与问题主题不匹配且系统可以识别单一缺口
- **那么** Sufficiency Gate 必须返回主题不匹配约束并附带 1 个候选澄清问题，而不是直接决定最终澄清姿态

#### 场景:相关性低且不可澄清时返回阻断约束
- **当** 证据主题与问题主题不匹配且无法确定有效澄清问题
- **那么** Sufficiency Gate 必须返回高严重度主题不匹配约束，并将最终 `refuse（拒答）` 决策留给上层 `planner / policy（规划器 / 策略层）`

### 需求:关键要素缺失必须触发不足
当问题要求关键要素（如数值、方法步骤、实验条件、时间/主体限定）而证据未覆盖时，Sufficiency Gate 必须产出“关键要素缺失”的结构化约束，禁止继续直接进入“可回答”路径。对于开放式总结问题，Sufficiency Gate 必须禁止无依据推断“必须包含 numeric（数值）”并触发误阻断；若仍有部分可追溯证据，则必须在约束对象中明确“允许部分回答”的信号，而不是默认直接拒答。

#### 场景:缺失数值或方法细节触发关键要素约束
- **当** 问题明确要求数值或方法细节，且 `top_n evidence（Top-N 证据）` 未提供对应要素
- **那么** Sufficiency Gate 必须返回标明缺失要素类型的结构化约束对象，并将最终用户姿态交给上层决定

#### 场景:开放式总结不得误触发 numeric 缺口
- **当** 用户输入主题化总结类问题且未请求具体指标
- **那么** Sufficiency Gate 不得仅因 numeric（数值）缺失直接形成阻断约束，必须优先允许可追溯总结或最小澄清候选

### 需求:系统必须优先输出可追溯部分回答
当证据不足以支持完整回答但存在部分可追溯证据时，Sufficiency Gate 必须返回“允许部分回答”的结构化信号并明确不确定边界；禁止默认把该场景压缩为硬拒答约束。

#### 场景:部分证据可用时提供部分回答信号
- **当** 关键问题仅部分要素有证据支持
- **那么** Sufficiency Gate 必须在约束对象中显式标记“允许部分回答”，并附缺失要素信息
