## ADDED Requirements

### 需求:前端必须提供三路 Stage LLM 设置面板
系统必须在单一管理面板中同时提供 `answer`、`embedding`、`rerank` 三个 stage 的配置编辑能力。每个 stage 至少必须包含 `provider`、`api_base`、`api_key` 与 `model` 输入项。

#### 场景:管理员打开三路设置面板
- **当** 管理员进入 LLM 连接设置区域
- **那么** 系统必须显示 answer/embedding/rerank 三个独立配置分组及各自必填字段

### 需求:前端必须提交三路配置载荷
当管理员保存配置时，前端必须提交包含 answer/embedding/rerank 三路对象的载荷到管理接口，禁止仅提交单一路径字段。

#### 场景:三路配置保存
- **当** 管理员完成三路配置并点击保存
- **那么** 系统必须向 `/api/admin/llm-config` 发送三路结构请求并展示保存结果

### 需求:前端必须正确回显三路持久化配置
前端读取管理接口返回时必须将 answer/embedding/rerank 三路配置回填到对应表单区域，禁止将多路配置压缩为单路展示。

#### 场景:刷新页面后读取配置
- **当** 管理员刷新页面并重新加载配置
- **那么** 系统必须在三个 stage 分组中回显最新持久化值
