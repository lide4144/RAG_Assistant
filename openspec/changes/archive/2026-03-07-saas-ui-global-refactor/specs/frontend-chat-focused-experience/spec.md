## ADDED Requirements

### 需求:系统必须提供以对话为中心的聊天主界面
系统必须移除聊天页内与模型连接相关的配置输入区域，聊天页必须仅承载会话内容与对话输入相关操作。

#### 场景:访问聊天页
- **当** 用户进入“对话问答”页面
- **那么** 页面必须不显示 Answer/Embedding/Rerank 配置表单

### 需求:系统必须支持 Markdown 聊天内容渲染
系统必须支持 Markdown 渲染，且必须支持代码高亮与公式渲染能力。

#### 场景:AI 返回带代码和公式的消息
- **当** AI 消息包含代码块与公式语法
- **那么** 系统必须以高亮代码与可读公式形式展示消息内容

### 需求:系统必须提供聊天空白状态
系统必须在无历史会话时展示居中品牌化空白态，引导用户开始提问。

#### 场景:首次进入无对话
- **当** 当前会话没有任何消息
- **那么** 页面必须显示空白态提示而不是空白容器

### 需求:系统必须在模型未配置时阻止误用并提供跳转入口
系统必须在聊天页顶部显示醒目告警，并提供一键跳转至“模型设置”页面。

#### 场景:模型未配置时尝试对话
- **当** 系统检测到模型配置状态为未完成
- **那么** 页面必须显示告警条并提供可点击跳转入口到“模型设置”

### 需求:系统必须提供可区分的聊天气泡视觉
系统必须对用户消息与 AI 消息使用可区分的毛玻璃卡片样式，保证信息分层清晰。

#### 场景:同屏展示用户与 AI 消息
- **当** 会话中同时存在用户与 AI 消息
- **那么** 两类消息气泡必须在视觉上可被快速区分
