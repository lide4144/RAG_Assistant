## ADDED Requirements

### 需求:系统必须提供全局 SaaS 导航壳层
系统必须提供固定的应用壳层，左侧为沉浸式导航栏，右侧为主内容区域，并必须在所有主业务页面保持一致布局结构。

#### 场景:进入应用首页
- **当** 用户打开前端应用
- **那么** 系统必须显示左侧导航与右侧主内容区，且默认落在“对话问答”页面

### 需求:系统必须提供三项主导航入口
系统必须在侧边导航中提供“对话问答”“知识处理”“模型设置”三项入口，禁止使用英文术语作为主标签。

#### 场景:导航可见性
- **当** 任意页面加载完成
- **那么** 用户必须始终可见上述三项导航入口

### 需求:系统必须通过路由实现无刷新页面切换
系统必须通过前端路由进行页面切换，禁止通过整页刷新进行导航。

#### 场景:从对话问答切换到模型设置
- **当** 用户点击“模型设置”导航项
- **那么** 系统必须在不刷新浏览器页面的前提下切换到设置页面并保留应用壳层

### 需求:系统必须在导航中标记当前页面状态
系统必须为当前激活导航项提供明显视觉状态，禁止出现“用户无法判断当前所在页面”的情况。

#### 场景:当前页面高亮
- **当** 用户位于“知识处理”页面
- **那么** “知识处理”导航项必须显示激活态且其余导航项显示非激活态
