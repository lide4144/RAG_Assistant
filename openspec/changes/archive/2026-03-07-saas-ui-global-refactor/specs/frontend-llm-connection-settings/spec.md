## ADDED Requirements

### 需求:系统必须提供模型连接设置的保存反馈
系统必须在“测试并保存”操作期间显示加载状态，并必须在保存成功后给出 Toast 成功提示。

#### 场景:保存成功反馈
- **当** 管理员点击“测试并保存”且后端返回成功
- **那么** 按钮必须显示加载结束状态并展示成功 Toast

### 需求:系统必须支持 Provider 预设联动
系统必须在管理员选择 Provider 后自动填充推荐 API Base，且必须允许管理员在自动填充后手动覆盖。

#### 场景:选择服务商
- **当** 管理员在 Provider 下拉框中选择某服务商
- **那么** 系统必须自动填充推荐 API Base 并允许后续修改

## MODIFIED Requirements

### 需求:系统必须提供 LLM 连接设置面板
系统必须在“模型设置”页面提供 stage 化设置面板，不仅包含基础连接参数，还必须支持 `answer`、`embedding`、`rerank` 三路独立配置输入，禁止仅提供单路配置表单。

#### 场景:管理员填写连接参数
- **当** 管理员进入“模型设置”页面
- **那么** 系统必须显示 answer、embedding、rerank 三组配置字段并允许独立编辑

### 需求:系统必须支持保存最终配置
系统必须提供“测试并保存”按钮，将三路配置一次性提交到后端配置持久化接口；在提交期间按钮必须显示加载状态，且系统必须保持对旧单路接口响应的兼容处理，避免 UI 崩溃。

#### 场景:完成配置保存
- **当** 管理员完成三路配置并点击“测试并保存”
- **那么** 系统必须提交三路配置并反馈保存成功状态

### 需求:系统必须提供可理解的错误反馈
系统必须根据后端错误码展示可理解提示，禁止仅显示无上下文的通用失败文案；对于密钥错误、网络失败、模型不可用等常见错误必须给出可执行指引。

#### 场景:鉴权失败提示
- **当** 后端返回 `AUTH_FAILED`
- **那么** 系统必须提示管理员检查 API Key 或权限并保持已填写字段不丢失

### 需求:系统必须提供安全的 API Key 输入体验
系统必须默认掩码显示 API Key 输入值，并必须提供显隐切换控件；禁止在非用户显式操作下明文展示密钥。

#### 场景:切换密钥可见性
- **当** 管理员点击 API Key 显隐图标
- **那么** 系统必须仅切换当前输入框的显示形态且不改变其实际值
