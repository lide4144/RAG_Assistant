## ADDED Requirements

### 需求:系统必须提供知识库构建流水线的统计总览区
系统必须在页面顶部提供统计卡片区，并必须使用网格化布局承载关键指标。

#### 场景:进入知识处理页
- **当** 用户进入“知识处理”页面
- **那么** 页面顶部必须显示统计总览区并展示关键指标数据

### 需求:系统必须提供动态数字反馈
系统必须为关键数字指标提供动态过渡展示，禁止在数据刷新时出现无过渡跳变。

#### 场景:统计指标更新
- **当** 流水线状态变化导致指标刷新
- **那么** 指标数字必须以可感知的过渡动画更新

### 需求:系统必须提供四阶段实时状态指示
系统必须对 Import、Clean、Index、Graph Build 四个阶段提供统一状态指示，并必须至少覆盖 running、success、failed 三类状态颜色语义。

#### 场景:阶段执行中
- **当** 某一阶段状态为运行中
- **那么** 该阶段必须显示脉冲状态点并采用运行中色彩

#### 场景:阶段失败
- **当** 某一阶段状态为失败
- **那么** 该阶段必须显示失败色并可被用户立即识别

### 需求:系统必须在存在图构建任务时展示终端风格日志区
系统必须在检测到 Graph Build 任务时展示终端风格日志流窗口，用于呈现实时或近实时任务日志。

#### 场景:Graph Build 任务存在
- **当** 当前任务集中包含 Graph Build 任务
- **那么** 页面底部必须显示终端风格日志窗口并持续追加日志内容
