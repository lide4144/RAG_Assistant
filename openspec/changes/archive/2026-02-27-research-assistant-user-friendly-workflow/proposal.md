## 为什么

当前系统主要面向开发调试与检索实验，对普通科研同学的使用门槛偏高：用户需要理解入库/索引/检索参数，难以在短时间内完成“收集论文->提问->沉淀灵感”的核心闭环。现在需要将产品重心从“RAG 实验台”调整为“个人论文知识库与科研助手”，以提升可用性、复用率与日常留存。

## 变更内容

- 重构面向用户的信息架构，统一为 `论文库 Library`、`灵感助手 Chat`、`灵感卡片 Ideas` 三个主视图。
- 提供普通用户默认路径：低门槛导入论文、直接提问、回答附可追溯证据、一键生成并保存灵感卡片。
- 将开发者调试能力（检索模式、图扩展参数、trace 细节等）下沉到高级设置，不作为默认交互。
- 引入灵感卡片生命周期管理（draft -> shortlisted -> in_progress -> validated -> archived），支持科研过程中的持续沉淀。
- 明确产品级验收标准：用户 2 分钟内完成“导入 5 篇 -> 提问 -> 保存 1 张灵感卡片”。

## 功能 (Capabilities)

### 新增功能
- `research-assistant-workbench`: 面向普通科研用户的工作台 IA 与导航结构，聚焦论文库、问答与灵感沉淀。
- `idea-cards-lifecycle`: 灵感卡片的数据模型、自动生成、可编辑保存与生命周期流转。
- `library-quick-ingestion`: 低门槛论文导入体验（以用户任务导向文案表达，不暴露底层流程术语）。

### 修改功能
- `paper-assistant-mode`: 从“可用但偏工程化”升级为“默认用户友好模式”，调整默认交互与输出组织。
- `visual-debug-interactive-ui`: 保留调试能力但默认隐藏在高级设置，避免干扰普通用户主流程。

## 影响

- 受影响代码：`app/ui.py`、`app/qa.py`、`app/session_state.py`、部分配置与文案映射逻辑。
- 受影响产物：会话输出结构与卡片持久化结构将扩展；需要与既有 `runs/*` 与 session store 兼容。
- 风险与约束：需保持当前证据可追溯与门控能力不退化，同时避免默认界面暴露过多工程参数。
