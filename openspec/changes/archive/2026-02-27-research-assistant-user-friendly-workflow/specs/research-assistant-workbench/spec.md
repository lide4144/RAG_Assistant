## 新增需求

### 需求:系统必须提供面向科研用户的三分区工作台导航
系统必须提供 `Library`、`Chat`、`Ideas` 三个一级工作区，并且默认进入 `Chat` 以支持快速提问。系统禁止将检索参数与调试字段作为默认首屏信息。

#### 场景:首屏进入可提问状态
- **当** 用户打开系统
- **那么** 界面必须在 1 次可见交互内提供提问输入框与当前专题上下文

### 需求:系统必须支持从回答到灵感沉淀的一键闭环
系统必须允许用户在回答完成后一键生成灵感卡片草稿，并在同一流程中完成保存。

#### 场景:回答后可直接生成卡片
- **当** 助手回答完成且存在可追溯证据
- **那么** 界面必须提供“生成灵感卡片”操作，并生成可编辑草稿

### 需求:系统必须提供任务导向术语
前台用户文案必须使用任务导向术语（例如导入论文、证据来源、灵感卡片），禁止默认展示 ingest/index/retrieval 等工程术语。

#### 场景:用户界面术语符合任务语义
- **当** 用户浏览主流程页面
- **那么** 页面中的主操作与提示必须采用任务导向术语
