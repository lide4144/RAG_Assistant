## 修改需求

### 需求:系统必须提供可运行的双区调试 UI
系统必须保留开发者审查面板能力，但默认在用户主流程中折叠或隐藏，避免干扰普通科研用户。

#### 场景:默认视图聚焦用户任务
- **当** 普通用户进入 Chat 页面
- **那么** 主界面必须优先展示对话与证据/卡片区，调试面板不作为默认主视觉

### 需求:系统必须支持回答引用的可交互审查
系统必须继续支持引用点击审查能力，并允许在用户友好视图中查看“证据来源”而不暴露底层冗余字段。

#### 场景:用户视图查看证据来源
- **当** 用户点击回答中的引用标记
- **那么** 系统必须展示可读证据来源（论文、位置、片段），且无需进入开发者模式
