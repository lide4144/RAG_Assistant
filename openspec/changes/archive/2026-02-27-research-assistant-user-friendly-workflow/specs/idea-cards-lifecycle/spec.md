## 新增需求

### 需求:系统必须定义灵感卡片最小数据结构
系统必须为每张灵感卡片保存 `title`、`research_question`、`method_outline`、`next_experiments` 与 `evidence`，并记录来源会话标识。

#### 场景:卡片结构完整写入
- **当** 用户保存灵感卡片
- **那么** 存储记录必须包含必填字段与来源信息，且可被再次读取展示

### 需求:系统必须支持卡片可编辑保存
系统生成的灵感卡片草稿必须允许用户在保存前修改关键字段。

#### 场景:用户修订后保存
- **当** 用户编辑卡片标题或研究问题并点击保存
- **那么** 系统必须保存编辑后的内容而非原始自动生成文本

### 需求:系统必须支持卡片生命周期流转
系统必须支持卡片状态 `draft -> shortlisted -> in_progress -> validated -> archived` 的可追踪流转。

#### 场景:状态从草稿推进
- **当** 用户将卡片从 `draft` 更新为 `shortlisted`
- **那么** 系统必须记录新的状态与更新时间
