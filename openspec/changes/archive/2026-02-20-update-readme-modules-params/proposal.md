## 为什么

当前 `README.md` 具备流程级说明，但对“每个模块如何单独使用、参数语义和取值影响”的覆盖不完整，导致新成员和后续维护者需要频繁查源码或反复试错。随着模块增多（ingest/clean/index/graph/qa/rewrite/evidence policy 等），文档缺少系统化参数解释会直接影响可用性、可复现性与协作效率。

## 变更内容

- 重构并扩展 README 的“模块化使用指南”，按模块拆分说明：用途、输入输出、典型命令、常见场景。
- 增加参数参考区，覆盖 CLI 参数与关键配置项（`configs/default.yaml`），说明默认值、推荐值、行为影响与常见误区。
- 增加跨模块关系说明，明确从入库到问答的依赖顺序和最小可运行路径。
- 增加排障与验证建议，帮助定位常见失败点（索引缺失、embedding 配置、graph 文件缺失、证据门控触发等）。
- 不修改应用行为和代码逻辑，仅更新文档结构与说明深度。

## 功能 (Capabilities)

### 新增功能
- 无

### 修改功能
- `project-readme-guide`: 扩展 README 文档规范，使其必须覆盖模块级使用方式与参数语义，包含命令示例、参数解释和排障建议。

## 影响

- 受影响文件：`README.md`
- 受影响规范：`openspec/specs/project-readme-guide/spec.md`
- 不涉及 API 变更、运行时依赖变更或数据格式变更。
