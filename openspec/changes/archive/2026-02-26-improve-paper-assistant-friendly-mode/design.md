## 上下文

当前版本已经具备助理优先、最小澄清、连续澄清上限等能力，但发布判断主要依赖一次性人工汇总，缺少固定问题集、可复现双策略对比和统一统计口径。为了支持增长优先目标，需要将评测协议产品化为“可重复执行的发布门禁”，并且与现有 trace 字段对齐。

## 目标 / 非目标

**目标：**
- 固化一套增长优先评测协议（问题集结构、分桶、统计口径、放行门槛）。
- 要求同一问题集下对比 `assistant_mode_force_legacy_gate=true/false` 两种策略。
- 统一计分只使用 `qa_report.decision` 与可追溯引用字段，避免口径冲突。
- 保留风险护栏（连续澄清、语料外误答、无引用强断言）并定义回滚阈值。

**非目标：**
- 本变更不重写问答核心算法或检索排序逻辑。
- 本变更不引入外部服务，不改变底层索引格式。
- 本变更不替代已有单测；它补充发布评测层。

## 决策

### 决策 1：新增独立能力 `paper-assistant-growth-evaluation`
- 选择：新增专门规范定义问题集、分桶、计分与报告契约。
- 备选：仅在 `paper-assistant-mode` 中追加几条验收语句。
- 取舍：独立能力更易维护评测生命周期，避免业务行为与评测契约耦合。

### 决策 2：计分口径统一为 `qa_report.decision`
- 选择：发布评测只读取 `qa_report.decision`、`answer_citations`、`clarify_*`、`final_refuse_source` 等字段。
- 备选：混合使用 `final_decision` 与 trace 子字段。
- 取舍：单一口径可避免指标歧义与版本漂移。

### 决策 3：采用“分桶 + 双策略”判定
- 选择：A 首问开放总结、B 多轮追问、C 控制意图混入、D 语料外四桶分别统计，并要求新策略相对旧策略有净提升。
- 备选：仅做全量平均指标。
- 取舍：分桶能防止单一高频场景掩盖高风险子场景。

### 决策 4：增长优先但设置硬护栏
- 选择：提高可追溯回答率门槛，同时设置语料外误答与连续澄清硬阈值。
- 备选：只追求低拒答率。
- 取舍：可用性增长必须与风险可控同时成立。

## 风险 / 权衡

- [风险] 问题集分布不代表真实用户，导致门槛偏乐观。  
  → 缓解措施：问题集按桶版本化，要求每次发布抽样更新并记录变更。

- [风险] 不同配置 run 混入同一统计导致误判。  
  → 缓解措施：评测必须记录配置快照，只统计配置一致 run。

- [风险] 过度追求增长导致语料外误答上升。  
  → 缓解措施：设置 OOC 强护栏与回滚触发条件。

## Migration Plan

1. 增加/更新规范：`paper-assistant-growth-evaluation`、`paper-assistant-mode`、`sufficiency-gate`。  
2. 在实现阶段补齐问题集文件、评测脚本和报告模板。  
3. 按固定问题集运行旧/新策略对比，生成发布报告。  
4. 若触发硬护栏（如 OOC 误答超阈值），切回 `assistant_mode_force_legacy_gate=true`。
