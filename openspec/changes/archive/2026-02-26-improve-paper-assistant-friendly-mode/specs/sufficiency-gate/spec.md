## 修改需求

### 需求:Sufficiency Gate 决策输出
系统必须在最终回答前基于 `question` 与 `top_n evidence` 产出单一决策对象，字段至少包含 `decision` 与 `reason`；其中 `decision` 必须为 `answer`、`refuse`、`clarify` 三者之一。当 `decision=clarify` 时，系统必须额外输出 `clarify_questions`，且数量必须为 1 个。系统必须输出可供 UI 审查的告警字段（含 `output_warnings` 或等价结构），并确保 `reason` 与告警字段可序列化且可直接展示。发布评测统计必须使用该决策对象中的 `decision` 作为唯一口径。

#### 场景:输出结构合法
- **当** 系统收到一次问答请求并完成证据聚合
- **那么** 系统必须返回包含 `decision` 与 `reason` 的决策对象，且 `decision` 取值仅限 `answer/refuse/clarify`

#### 场景:clarify 输出问题数量固定为 1
- **当** Sufficiency Gate 判定为 `clarify`
- **那么** 输出必须包含 `clarify_questions`，且问题数量必须恰好为 1

#### 场景:发布统计口径唯一
- **当** 系统执行增长评测汇总
- **那么** 统计必须仅基于 `qa_report.decision`，不得以 `final_decision` 等其它字段替代

### 需求:主题不匹配必须触发不足
当 `top_n evidence` 与问题主题相关性低于门限时，系统必须判定为证据不足，且禁止直接进入 `answer` 决策。对于可通过单一澄清补齐主题约束的场景，系统必须优先输出 `clarify` 而非 `refuse`；仅当无法形成有效澄清路径时才允许 `refuse`。对于控制意图轮次，系统必须使用锚定主题查询（`anchor_query` 或等价字段）作为主题匹配输入来源，而非直接使用控制语句本身。

#### 场景:相关性低但可澄清时优先澄清
- **当** 证据主题与问题主题不匹配且系统可以识别单一缺口
- **那么** 系统必须输出 `clarify`，并在 `reason` 中说明缺失主题约束

#### 场景:相关性低且不可澄清时拒答
- **当** 证据主题与问题主题不匹配且无法确定有效澄清问题
- **那么** 系统必须输出 `refuse`，并在 `reason` 中说明“主题不匹配/相关性不足”

#### 场景:助理模式下不可连续两轮模板化拒答
- **当** `assistant_mode_used=true` 且存在可引用证据
- **那么** 系统不得在相邻两轮都输出模板化拒答，至少一轮必须进入可追溯回答或最小澄清
