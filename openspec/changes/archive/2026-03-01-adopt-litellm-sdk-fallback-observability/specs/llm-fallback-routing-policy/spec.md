## ADDED Requirements

### 需求:系统必须提供可配置的主备模型降级策略
系统必须支持为每个调用阶段配置主模型与备模型，并在主模型出现可恢复错误时自动降级，禁止依赖供应商名称或 URL 字符串判断降级路径。

#### 场景:主模型超时触发备模型
- **当** 主模型请求超时且命中可恢复错误类别
- **那么** 系统必须按配置切换到备模型继续尝试并返回最终结果

### 需求:系统必须支持 retry 与 cooldown 策略
系统必须支持按模型或路由配置重试次数与冷却时间，并在连续失败阈值达到后暂时熔断对应模型。

#### 场景:连续失败触发冷却
- **当** 某模型在窗口期内连续失败达到阈值
- **那么** 系统必须在冷却窗口内停止优先路由到该模型
