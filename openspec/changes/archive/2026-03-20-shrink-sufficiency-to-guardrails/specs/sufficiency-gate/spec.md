## 新增需求

### 需求:系统必须将证据语义判别与硬校验分层
系统必须将 `Sufficiency Gate（充分性门控）` 拆分为两类受限职责：一类是独立的 `semantic evidence judge（语义证据判别）` 小模型，负责判断当前证据是否回答问题、是否仅支持部分回答、以及仍缺失哪些方面；另一类是 `deterministic validator（确定性校验器）`，负责最小证据存在性、证据类型噪声、引用/证据合法性与明显越界阻断。禁止继续把这两类职责混杂为一组静态规则词表和阈值裁决链。

#### 场景:语义判别与硬校验来源可区分
- **当** 系统完成一次证据充分性检查
- **那么** 输出中必须能够区分哪些结论来自 `semantic evidence judge`，哪些结论来自 `deterministic validator`

#### 场景:judge 调用失败时不得回落到本地启发式裁决
- **当** `semantic evidence judge` 小模型调用失败、超时、缺少密钥或返回非法载荷
- **那么** 系统必须在重试后返回高严重度系统错误或等价阻断约束，不得回落到本地启发式 / 词法规则路径继续裁决

### 需求:系统必须禁止用静态问题类型和槽位规则主导充分性裁决
系统必须禁止继续使用静态 `question type（问题类型）` 词表、`key element（关键要素）` 槽位映射或同构硬编码规则，作为 `topic mismatch（主题不匹配）`、`missing aspects（缺失方面）` 或“是否可回答”的正式主裁决依据。若语义判别无法给出稳定结果，系统必须进入受控不确定语义或系统错误阻断，而不是退回旧槽位规则链。

#### 场景:隐式问法不再依赖词表命中
- **当** 用户以未命中既有词表的方式询问实验结果、方法机制或限定条件
- **那么** 系统必须仍由语义证据判别得出覆盖结论，而不得因为未命中静态词表就默认进入旧规则裁决

#### 场景:`qa.py` 中的同构词表不得继续作为旁路主裁决
- **当** `qa.py` 中存在与 `sufficiency gate` 同构的 `KEY_ELEMENT_*`、`QUESTION_TYPE_*` 或等价静态词表
- **那么** 系统必须删除、降权或显式隔离这些词表，禁止其继续作为证据充分性判断的旁路正式依据

### 需求:系统必须以结构化缺失方面替代旧关键要素槽位
系统必须使用结构化 `missing_aspects（缺失方面）`、`coverage_summary（覆盖摘要）` 与必要的证据覆盖说明来表达不足原因，而不是继续暴露或依赖静态 `missing_key_elements（缺失关键要素）` 槽位名。上层 `planner / policy（规划器 / 策略层）` 必须消费这些结构化语义结果，而不是继续绑定旧槽位枚举。

#### 场景:部分回答场景输出覆盖摘要
- **当** 当前证据仅支持问题中的部分方面
- **那么** 系统必须输出结构化 `coverage_summary` 与 `missing_aspects`，并显式标记是否允许部分回答

## 修改需求

### 需求:Sufficiency Gate 决策输出
系统必须将 `Sufficiency Gate（充分性门控）` 定义为 `guardrail（护栏）` 组件，而不是最终用户交互裁判。Sufficiency Gate 必须在回答前基于 `question（问题）` 与 `top_n evidence（Top-N 证据）` 产出结构化约束对象，至少包含 `reason（原因）`、`reason_code（原因代码）`、`severity（严重度）`、`clarify_questions（澄清问题，可选）`、`output_warnings（告警）`、`missing_aspects（缺失方面）`、`coverage_summary（覆盖摘要）` 以及必要的 judge/validator 来源字段。禁止要求其以 `answer / refuse / clarify（回答 / 拒答 / 澄清）` 作为唯一顶层决策口径；最终用户可见姿态必须由 `planner / policy（规划器 / 策略层）` 基于该约束对象统一决定。

#### 场景:输出结构包含来源与覆盖摘要
- **当** 系统收到一次问答请求并完成证据聚合
- **那么** Sufficiency Gate 必须返回带有 judge/validator 来源字段、覆盖摘要和缺失方面的结构化约束对象，而不得直接作为最终用户姿态裁判

### 需求:主题不匹配必须触发不足
当 `top_n evidence（Top-N 证据）` 与问题语义上不相符时，Sufficiency Gate 必须产出“主题不匹配”或等价的结构化约束，并禁止将其静默视为可直接回答。该判断必须优先来自 `semantic evidence judge（语义证据判别）` 或等价语义能力，而不得以 token overlap、trigram 相似度或同构近似规则作为正式主裁决依据。对于控制意图轮次，Sufficiency Gate 必须使用锚定主题查询（`anchor_query（锚定查询）` 或等价字段）作为语义判别输入来源，而非直接使用控制语句本身。

#### 场景:相关性不足由语义判别得出
- **当** 证据与问题主题不相符但文本表面重合度并不足以说明真实语义关系
- **那么** 系统必须优先依据语义证据判别结果产出不足约束，而不得仅依赖 token overlap / trigram 规则作出正式结论

### 需求:系统必须优先输出可追溯部分回答
当证据不足以支持完整回答但存在部分可追溯证据时，Sufficiency Gate 必须返回“允许部分回答”的结构化信号并明确不确定边界。该判断必须基于语义覆盖摘要，而不是继续通过静态数值/方法/主体等槽位表推断“必须缺某个 key element（关键要素）”。

#### 场景:开放式总结不再被静态 numeric 规则误伤
- **当** 用户提出开放式总结或高层归纳类问题，且证据未提供具体数值
- **那么** 系统不得仅因未命中静态 numeric（数值）槽位规则就形成阻断约束，必须优先依据语义覆盖摘要决定是否允许可追溯部分回答
