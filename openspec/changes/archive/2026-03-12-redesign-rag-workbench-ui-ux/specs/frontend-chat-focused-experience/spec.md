## 新增需求

### 需求:系统必须提供历史会话侧栏
系统必须在聊天页提供独立的历史会话侧栏，至少包含“+ 新建对话”入口，以及按“今天”“昨天”“更早”分组的历史会话列表。系统必须允许用户从历史列表切换回之前的会话内容，并必须支持删除不再需要的旧会话，禁止聊天页仅支持单次临时会话。

#### 场景:查看历史会话
- **当** 用户进入聊天页且浏览器存在历史会话记录
- **那么** 页面必须按时间分组显示历史会话，并支持点击切换

#### 场景:新建会话
- **当** 用户点击“+ 新建对话”
- **那么** 系统必须清空当前会话展示区并创建新的会话上下文

#### 场景:删除旧会话
- **当** 用户在历史会话列表中删除某个旧会话
- **那么** 系统必须将该会话从历史列表移除，且删除当前激活会话时必须自动切换到剩余会话或创建新的空白会话

## 修改需求

### 需求:系统必须提供以对话为中心的聊天主界面
系统必须将聊天页组织为“历史会话侧栏 + 主对话区 + 会话辅助信息区”的结构。聊天页必须优先突出提问、回答、引用和继续追问流程，禁止只提供单列无上下文的会话容器。

#### 场景:进入聊天页
- **当** 用户打开“对话问答”页面
- **那么** 页面必须显示历史会话入口、主对话区和与当前会话相关的辅助摘要

### 需求:系统必须提供聊天空白状态
系统必须在无历史消息时展示中文引导文案和快捷提问卡片，并必须说明系统会给出证据来源。快捷卡片至少保留 3 个，且必须支持点击后直接发送。

#### 场景:首次进入无对话
- **当** 当前会话没有任何消息
- **那么** 页面必须展示中文引导和可点击的快捷提问卡片

### 需求:系统必须提供可区分的聊天气泡视觉
系统必须继续区分用户消息与 AI 消息，但必须升级为更明确的卡片式层级，并与引用面板、图谱面板保持一致的中文友好视觉风格。

#### 场景:同屏展示用户与 AI 消息
- **当** 会话中同时存在用户与 AI 消息
- **那么** 两类消息必须通过背景、位置和标签被快速区分
