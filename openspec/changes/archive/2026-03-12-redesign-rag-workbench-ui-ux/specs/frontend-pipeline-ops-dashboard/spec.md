## 新增需求

### 需求:系统必须提供导入流程总进度反馈
系统必须在知识处理页的“处理状态”区域提供总体进度反馈，包括当前阶段说明、百分比进度条或等效的已完成比例展示。系统禁止仅展示彼此孤立的阶段状态卡而不说明整体流程已经推进到哪里。

#### 场景:导入后查看总体进度
- **当** 用户触发导入或图构建相关处理
- **那么** 页面必须展示当前环节、进度比例以及对应说明

## 修改需求

### 需求:系统必须提供四阶段实时状态指示
系统必须以中文可读方式展示“导入 -> 清洗 -> 索引 -> 图构建”四阶段流程，并必须将当前激活阶段高亮。系统必须允许阶段总览和图构建细粒度进度同时存在，禁止仅显示“待处理”“未知”等孤立文本而不说明所处流程位置。

#### 场景:步骤流程推进
- **当** 阶段状态更新
- **那么** 页面必须高亮当前阶段，并在同一区域展示阶段标签、时间和状态说明

#### 场景:图构建仍在执行
- **当** 图构建任务仍在运行
- **那么** 页面必须同时显示流程阶段位置和图构建已处理数量或百分比

### 需求:系统必须提供知识库构建流水线的统计总览区
系统必须继续在顶部提供关键统计卡片，但必须与导入入口和处理状态形成更清晰的信息分层，便于中文用户先看结果总览，再看当前任务进度。

#### 场景:进入知识处理页
- **当** 用户进入“知识处理”页面
- **那么** 页面必须先展示统计总览，再展示导入入口和处理状态
