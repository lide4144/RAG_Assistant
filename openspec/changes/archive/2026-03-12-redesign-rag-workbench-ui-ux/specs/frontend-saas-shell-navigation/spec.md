## 修改需求

### 需求:系统必须提供全局 SaaS 导航壳层
系统必须将全局壳层升级为统一的 SaaS 工作台容器，必须同时提供中文品牌区、主线说明、三项主导航和运行态状态条。壳层在加载运行态摘要时，必须继续通过部署友好的 `/api/admin/*` 路径读取数据，禁止因视觉重构破坏现有管理接口寻址策略。

#### 场景:进入任一主页面
- **当** 用户打开“对话问答”“知识处理”或“模型设置”任一页面
- **那么** 系统必须展示统一的中文导航壳层、品牌说明和三项主导航入口

#### 场景:运行态正常时显示轻量状态条
- **当** 壳层成功读取运行态摘要
- **那么** 系统必须以轻量状态条显示连接与运行态信息，而不是按钮式控件或原始技术枚举堆叠

### 需求:系统必须在导航中标记当前页面状态
系统必须以更清晰的视觉高亮标记当前页面，并必须将连接状态显示为不可点击的中文指示器；连接成功时不得再依赖英文或表情作为唯一语义。

#### 场景:当前页面为聊天页
- **当** 用户位于“对话问答”页面
- **那么** 导航必须高亮“对话问答”，并保持其他导航可无刷新切换

#### 场景:连接成功
- **当** 前端与后端连通
- **那么** 壳层必须显示中文“已连接”状态和绿色状态点
