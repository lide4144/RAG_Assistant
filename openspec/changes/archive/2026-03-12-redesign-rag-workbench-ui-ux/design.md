## 上下文

当前工作台的前端能力已经覆盖聊天、知识处理、模型配置和生成文件管理，但页面结构仍然偏技术内核视角：同一页面同时暴露底层参数、运行态枚举和文件级术语，导致普通中文用户理解门槛高。此次变更涉及全局壳层、聊天页、知识处理页、模型设置页和生成文件面板，是一次跨多个前端模块的信息架构重排，而不是局部样式修补。

现有实现约束如下：
- 保持现有接口契约不变，继续复用 `runtime-overview`、导入结果、任务事件和配置保存接口
- 延续 Next.js + React + Tailwind 现有技术栈，不新增重量级 UI 依赖
- 保持已有 `StatusMapper`、`fetchAdminJson`、部署友好路由策略不变
- 允许在前端本地存储聊天历史，但不要求新增后端持久化接口

## 目标 / 非目标

**目标：**
- 为三大核心页面建立统一的中文友好 SaaS 视觉语言
- 让知识处理页明确展示“导入 -> 清洗 -> 索引 -> 图构建”的阶段进度
- 让聊天页具备可回看的历史对话入口和更清晰的会话信息层
- 让模型设置页优先暴露高频可操作项，并把晦涩参数折叠收纳
- 在不改变后端 API 的前提下完成交互重构

**非目标：**
- 不新增新的后端聊天历史存储接口
- 不修改 Kernel/Gateway 返回的数据结构
- 不引入新的设计系统或第三方组件框架迁移
- 不处理与本次 UI/UX 无关的检索、推理或索引算法变更

## 决策

### 决策 1：保留现有数据流，重构展示层
- 选择：继续沿用当前 `fetchAdminJson`、WebSocket 和状态映射逻辑，只调整页面布局、文案和控件层级
- 原因：本次问题核心是可理解性和反馈不足，不是接口能力缺失；直接重构展示层更快且风险更低
- 备选方案：
  - 先改后端返回结构：收益有限，且会扩大变更面
  - 新建平行页面：会造成规范和实现重复

### 决策 2：聊天历史采用前端本地持久化
- 选择：在聊天页使用 `localStorage` 保存会话摘要和消息快照，支持“今天 / 昨天 / 更早”分组
- 原因：用户当前需求集中在“找回之前对话”，本地持久化已足够覆盖单端使用场景，无需等待后端新增能力
- 备选方案：
  - 新增后端会话接口：更完整，但超出本轮范围
  - 仅做静态侧栏：无法满足真实可回看需求

### 决策 3：知识处理页使用“总进度 + 当前环节 + 四段步骤卡”
- 选择：在处理状态区组合进度条、当前阶段摘要和四个流程阶段卡，而不是继续使用并列静态状态盒子
- 原因：用户关心“现在到哪一步了”，该结构比原始阶段盒子更符合任务型流程认知
- 备选方案：
  - 仅增加百分比：不能明确说明具体环节
  - 仅增加 stepper：对图构建阶段的细粒度进度表达不足

### 决策 4：模型设置拆为“核心模型选择 + 高级模型调优”
- 选择：把回答模型、重排模型、问题改写模型提升到概览区直接可选，把 Marker Runtime Tuning 放入默认折叠的 `details/summary`
- 原因：高频操作应该优先可见，低频底层参数不应压制主要任务
- 备选方案：
  - 保留当前单屏全展开：继续制造信息噪音
  - 完全隐藏高级参数：会损失专业运维能力

### 决策 5：统一中文文案并对必要术语补 Tooltip
- 选择：将 `Artifact`、`Import path healthy`、`Rerank` 等生硬术语替换为中文含义，并用轻量 Tooltip 解释难以完全翻译的专业项
- 原因：单纯中英混排会让非技术用户误以为自己进入了调试台，而非产品界面
- 备选方案：
  - 全保留英文：不满足本次本地化目标
  - 全部硬翻译：会损失部分专业精度

## 风险 / 权衡

- [聊天历史仅保存在浏览器本地] → 通过明确限定为前端会话回看能力，避免误导为跨设备同步
- [增量重构多个页面，样式改动面较大] → 保持接口和页面路由不变，只替换组件结构和文案层
- [Tooltip 与折叠面板可能带来可访问性细节遗漏] → 使用原生 `title`/轻量悬浮提示和 `details/summary` 优先保证基础可用
- [进度百分比与阶段完成数来自不同数据源] → 允许阶段总览与图构建细进度并存，优先表达“当前在做什么”

## Migration Plan

- 第一步：更新全局样式变量和壳层布局，建立统一视觉语言
- 第二步：重构聊天页并加入本地历史会话存储
- 第三步：重构知识处理页的导入区、处理状态区和生成文件文案
- 第四步：重构模型设置页，拆分核心选择与高级调优面板
- 回滚策略：任何页面出现回归时，可按组件粒度回退 `chat-shell`、`PipelineWorkbenchPanel`、`settings-shell`、`app-shell`、`marker-artifact-panel`
