## 修改需求

### 需求:系统必须在模型未配置时阻止误用并提供跳转入口
聊天页在读取运行态摘要（`/api/admin/runtime-overview`）时，必须具备与壳层一致的路由稳定性和响应容错。接口异常时，系统必须提供可读错误提示并保持聊天输入区与导航的可操作性。

#### 场景:运行态接口地址缺失
- **当** 前端未显式配置 `NEXT_PUBLIC_KERNEL_BASE_URL`
- **那么** 聊天页运行态请求必须仍能命中 Kernel 服务，而不是 Next 前端 404 页面

#### 场景:运行态接口返回非 JSON
- **当** 运行态接口返回 HTML 或其他非 JSON 内容
- **那么** 系统必须显示“运行态概览不可用，请检查内核服务”等提示，且不得出现 JSON 解析异常堆栈
