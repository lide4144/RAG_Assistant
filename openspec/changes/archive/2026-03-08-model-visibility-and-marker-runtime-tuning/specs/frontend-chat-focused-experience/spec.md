## 修改需求

### 需求:系统必须在模型未配置时阻止误用并提供跳转入口
系统必须在聊天页展示“当前会话模型摘要”，至少包含 `answer`、`rerank`、`rewrite` 的生效模型信息与配置状态。系统不得仅以单一布尔值提示“已配置/未配置”而缺少具体模型上下文。

#### 场景:聊天页展示模型摘要
- **当** 用户进入“对话问答”页面
- **那么** 页面必须显示当前会话使用的关键模型摘要（provider/model）

#### 场景:关键模型缺失时阻止发送
- **当** `answer` 路由未配置或状态为 `BLOCKED`
- **那么** 系统必须阻止发送并显示可执行指引与跳转入口
