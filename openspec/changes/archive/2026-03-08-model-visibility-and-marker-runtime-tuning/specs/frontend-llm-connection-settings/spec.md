## 修改需求

### 需求:系统必须提供 LLM 连接设置面板
系统必须在“模型设置”页面中同时提供“配置编辑区”和“生效状态概览区”。概览区必须显示各 stage 的当前生效 provider/model 与状态，不得仅作为输入表单页面。

#### 场景:进入设置页查看生效状态
- **当** 管理员进入“模型设置”页面
- **那么** 页面必须展示各 stage 当前生效模型摘要与状态等级

### 需求:系统必须支持保存最终配置
系统必须支持在模型设置页中编辑并保存 Marker/Surya 运行档位参数，包括 `recognition_batch_size`、`detector_batch_size`、`layout_batch_size`、`ocr_error_batch_size`、`table_rec_batch_size`、`model_dtype`，并在保存后可回显。

#### 场景:保存 marker tuning 配置
- **当** 管理员修改 Marker 运行档位并点击保存
- **那么** 系统必须完成参数校验、持久化并回显最新生效值

#### 场景:参数越界提示
- **当** 管理员提交非法批大小或不支持的 dtype
- **那么** 系统必须给出字段级错误提示并保留其他已编辑输入
