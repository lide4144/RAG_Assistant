## 修改需求

### 需求:系统必须提供全局 SaaS 导航壳层
系统必须在全局壳层中提供“模型运行状态条”，用于跨页面展示当前关键 stage 的生效模型摘要与运行状态等级（`READY`/`DEGRADED`/`BLOCKED`/`ERROR`）。状态条必须可在桌面与移动端可见，并提供跳转到“模型设置”页面的入口。

#### 场景:任意页面查看全局运行状态
- **当** 用户位于对话问答、知识处理或模型设置任一页面
- **那么** 系统必须显示一致的模型运行状态摘要与状态等级

#### 场景:状态异常时快速跳转
- **当** 全局状态等级为 `DEGRADED`、`BLOCKED` 或 `ERROR`
- **那么** 状态条必须提供一键跳转“模型设置”入口
