## 上下文

当前流程在检索与证据聚合后直接进入回答生成，虽有证据不足降级机制，但对“是否足够回答当前问题”的判定标准不够结构化，导致语料库外问题或细节缺失问题可能出现过度回答。M8 需要在最终回答前增加独立 Sufficiency Gate，统一输出 `answer` / `refuse` / `clarify` 决策，并将其接入最终响应编排。

约束：
- 输入固定为 `question` + `top_n evidence`，不引入额外外部检索。
- 结果必须可解释：输出短句 `reason`，便于审计与回归。
- 判定逻辑优先规则化，必要时可叠加轻量 LLM 判断，但不得弱化“证据不足不胡说”原则。

## 目标 / 非目标

**目标：**
- 新增 Sufficiency Gate，稳定输出 `{decision, reason, clarify_questions?}`。
- 明确两类不足判定并可触发：主题不匹配、关键要素缺失。
- 在 `clarify` 决策下生成 1~2 个澄清问题，问题应直接指向缺失信息。
- 保证语料库外问题默认拒答，完成 10 个构造样例的拒答回归。

**非目标：**
- 不在本里程碑内优化检索召回算法本身。
- 不引入新的外部知识源或联网补充事实。
- 不改变上游 chunk 构建与排序协议，仅消费既有 `top_n evidence`。

## 决策

### 决策 1：采用“两阶段充分性判断”
- 方案：先做规则门控（相关性阈值、关键槽位覆盖），再输出最终决策；仅在边界样本可选用 LLM 辅助解释。
- 选择理由：规则门控可解释、稳定、便于验收；边界场景可通过 LLM 生成更自然的 `reason/clarify_questions`。
- 备选方案：纯 LLM 分类（answer/refuse/clarify）。未选原因：波动较大且难保证“语料库外问题拒答率”。

### 决策 2：决策对象前置并驱动下游输出分支
- 方案：在回答生成前产出 `decision`，由 orchestrator 按分支执行：
  - `answer`: 进入正常回答路径；
  - `refuse`: 进入拒答模板路径；
  - `clarify`: 输出澄清问题并暂停直接回答。
- 选择理由：确保拒答/澄清不会被默认回答分支覆盖。
- 备选方案：在回答后做后置裁剪。未选原因：无法阻断编造内容先产生。

### 决策 3：关键要素缺失以“问题类型槽位”定义
- 方案：按问题类型（定义型、数值比较型、方法机制型、事实查证型）维护最小槽位要求，证据未覆盖槽位则触发不足。
- 选择理由：比固定关键词更稳健，能覆盖“缺数字”“缺方法细节”等验收点。
- 备选方案：仅用 evidence 条数阈值。未选原因：数量足够不代表信息充分。

## 风险 / 权衡

- [风险] 规则过严导致本可回答问题被拒答。  
  → 缓解：对边界样本记录判定特征并调参，保留人工抽检集。

- [风险] 澄清问题过泛，用户仍无法补充有效信息。  
  → 缓解：澄清问题模板绑定缺失槽位，限制为 1~2 个且可操作。

- [风险] 与既有 `证据不足降级` 逻辑重叠，行为冲突。  
  → 缓解：以 Sufficiency Gate 为唯一入口信号，下游仅消费 `decision`。

## Migration Plan

1. 在回答编排前插入 `run_sufficiency_gate(question, top_n_evidence)`。
2. 扩展输出 schema：新增 `decision`、`reason`、`clarify_questions`（可选）。
3. 改造回答编排分支，确保 `refuse/clarify` 不进入正常答案生成。
4. 新增测试：
   - 两类 insufficiency 触发单测；
   - 10 个语料库外问题拒答回归；
   - `clarify` 仅输出 1~2 个问题的格式检查。
5. 灰度观察拒答率与误拒答率，必要时调节阈值与槽位映射。

回滚策略：关闭 Sufficiency Gate 开关并回退到原回答编排；保留新增字段但允许为空。
