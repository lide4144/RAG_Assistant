## ADDED Requirements

### 需求:系统必须提供 GraphRAG 子图可视化
系统必须在 UI 中展示与当前问题相关的检索子图，至少包含 seed 节点、扩展节点与边类型；禁止仅展示 `graph_expand` 文本标签作为唯一图扩展反馈。

#### 场景:命中图扩展后显示子图
- **当** 当前轮检索包含 `graph_expand` 来源
- **那么** UI 必须展示对应子图并标注各节点来源类型

### 需求:系统必须支持图节点与引用联动
系统必须支持图节点与回答引用双向联动：点击引用可高亮节点，点击节点可定位证据与回答片段。

#### 场景:引用驱动图定位
- **当** 用户点击回答中的某条引用
- **那么** 子图中对应节点与路径必须高亮，并展示关联证据摘要

### 需求:系统必须限制可视化开销
系统必须对默认可视化规模设置上限（如 top-N 节点）并支持按需展开，禁止在默认视图渲染全图导致交互阻塞。

#### 场景:大图场景默认不阻塞
- **当** 检索子图节点数超过默认上限
- **那么** 系统必须仅渲染可视化子集并提供展开操作
