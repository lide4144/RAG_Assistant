## 新增需求

### 需求:项目 README 基础结构
系统必须在项目根目录维护 `README.md`，并且该文档必须包含“项目概览、已实现里程碑、快速开始、关键命令示例、常见问题/排错”章节。

#### 场景:新用户首次访问仓库
- **当** 用户打开项目根目录的 `README.md`
- **那么** 用户必须能够看到完整的章节结构并了解项目已实现能力

### 需求:按里程碑组织的使用说明
系统必须在 `README.md` 中按里程碑（至少 M1、M1.5、M2、M2.1、M3）说明输入、输出与最小可执行命令示例。

#### 场景:用户按里程碑执行流程
- **当** 用户参考 README 执行某里程碑命令
- **那么** 文档必须明确该命令的输入路径与输出产物位置

### 需求:示例命令可复用
系统必须在 README 中提供可直接复用的 CLI 示例，并禁止仅给出抽象描述而不提供命令。

#### 场景:用户复制示例命令
- **当** 用户从 README 复制命令执行
- **那么** 用户必须能够定位该命令对应的脚本入口与产物路径
