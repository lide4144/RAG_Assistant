## 为什么

当前 M3 的规则改写链路在实际问题上存在几个直接影响召回质量的问题：中文同义扩展难以命中、复合术语被拆分、关键词预算被噪声词占用，以及扩展命中标记不够准确。这会导致“有改写但效果不稳定”，与 M3 提升可检索性的目标不一致。

## 变更内容

- 强化 token 提取规则，支持中文词段与复合术语（如 `GUESS-18`、`PLA/BPST`、`Top-1`）。
- 强化同义扩展触发逻辑：
  - 支持中文词典键在中文词段中的子串命中。
  - 对英文复数做轻量归一（仅在词典键存在时生效）。
- 增加最小停用词过滤，避免关键词预算被无效词占满。
- 将 `keyword_expansion` 命中判定改为“是否实际新增扩展词”。
- 补充对应单元测试，覆盖中文扩展与复合术语场景。

## 功能 (Capabilities)

### 新增功能
- 无

### 修改功能
- `query-rewriting`: 调整规则改写的关键词抽取与扩展策略，提升中文/复合术语场景下的可检索性与可解释性。

## 影响

- 受影响代码：`app/rewrite.py`、`tests/test_rewrite.py`。
- 受影响行为：M3 改写产物中的 `keywords_entities` 与 `strategy_hits` 更稳定，召回前置查询更贴近问题实体与术语。
- 无新增外部依赖，无 API 破坏性变更。
