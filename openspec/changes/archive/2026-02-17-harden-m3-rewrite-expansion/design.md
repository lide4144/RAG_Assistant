## 上下文

M3 改写链路已经具备术语保留、同义扩展、问句去冗余与可选 LLM 开关，但在真实查询上暴露出稳定性问题：
- 中文 query 难以触发同义词扩展；
- 复合术语（`GUESS-18`、`PLA/BPST`）在 token 化中被拆分；
- 关键词预算受噪声词影响，扩展词进入率低；
- `keyword_expansion` 命中标记与真实扩展增量不完全一致。

本变更限定在 `app/rewrite.py` 与其单测，不引入外部依赖，目标是在保持规则链路轻量的前提下提升命中率与可解释性。

## 目标 / 非目标

**目标：**
- 提升关键词抽取对中文和复合术语的覆盖率。
- 提升同义扩展触发成功率（尤其中文与英文复数变体）。
- 降低关键词集合噪声，保证有限预算优先容纳高价值词。
- 使 `keyword_expansion` 标记与“是否新增扩展词”一致。
- 增加回归测试覆盖新增行为。

**非目标：**
- 不引入中文分词库、NLP 词形还原库或向量模型。
- 不调整 LLM 改写策略本身。
- 不改动检索/排序模块（仅改写阶段）。

## 决策

### 决策 1：扩展 token 正则以支持中文和复合术语
- 方案：`TOKEN_RE` 改为同时匹配 `A-B`、`A/B` 和中文词段。
- 原因：最小改动即可覆盖已发现的主要失败样例。
- 备选：引入分词器/更复杂 lexer。
  - 放弃原因：超出 M3 范围，增加依赖与维护成本。

### 决策 2：同义词扩展采用“词典驱动的轻量归一 + 中文子串命中”
- 方案：
  - 同义词 key 统一小写；
  - 仅当复数归一结果存在于词典 key 时才使用（避免过度截断）；
  - 对中文 token 允许命中词典中文 key 子串（如 `这篇论文的引用` 命中 `引用`）。
- 原因：在不引入分词库情况下显著提高扩展触发率。
- 备选：通用词形还原与中文分词。
  - 放弃原因：依赖重、收益与复杂度不成比例。

### 决策 3：加入最小停用词过滤保护关键词预算
- 方案：增加小规模中英停用词集合，在关键词入池前过滤。
- 原因：避免 `rewrite_max_keywords` 被低信息词占满导致扩展词丢失。
- 备选：大规模停用词词表。
  - 放弃原因：当前阶段维护成本高且易误杀领域词。

### 决策 4：`keyword_expansion` 仅在真实新增词时命中
- 方案：扩展函数返回 `expanded: bool`，由真实新增计数驱动 `strategy_hits`。
- 原因：日志语义精确，便于后续诊断与评估。
- 备选：继续以长度差推断。
  - 放弃原因：受预处理/去重影响，误报概率较高。

## 风险 / 权衡

- [风险] 中文子串命中可能引入少量误匹配  
  → 缓解：只对词典中显式中文 key 触发，且保留 `rewrite_max_keywords` 上限。

- [风险] 停用词表过小仍会漏噪声，过大可能误删有效词  
  → 缓解：先采用最小集合并通过测试和样例迭代。

- [风险] 复数归一规则简单，存在边缘词形未覆盖  
  → 缓解：仅在词典 key 存在时启用归一，后续按词典扩充。

## 迁移计划

1. 修改 `app/rewrite.py`：token 规则、扩展逻辑、停用词过滤、命中标记。
2. 增补 `tests/test_rewrite.py`：中文扩展、复合术语保持场景。
3. 运行相关单测（rewrite + 受影响模块）确认无回归。
4. 若出现误匹配，再按样例微调停用词与中文子串规则。

回滚策略：若线上效果异常，可回滚 `app/rewrite.py` 到变更前版本，单测将恢复到旧行为基线。
