## 上下文

当前系统由 Python FastAPI 内核与 Next.js 前端组成，LLM 相关配置主要来自 `configs/default.yaml` 与环境变量。现状中缺少管理员可视化配置能力，无法在前端验证第三方 OpenAI-compatible 端点可用模型，也无法将最终选择稳定写入可被运行时读取的配置层。

该变更跨越后端 API、前端交互、配置持久化和 LiteLLM 路由适配，属于跨模块改造。约束如下：
- 不破坏现有聊天接口与默认配置回退路径。
- 对外模型探测必须以后端代理方式进行，避免浏览器直连跨域与密钥泄露。
- 持久化层必须支持最小化改动接入当前 Router 读取链路。

## 目标 / 非目标

**目标：**
- 提供 `POST /api/admin/detect-models`，完成 OpenAI-compatible `/models` 探测、标准化返回与可操作错误分类。
- 提供前端设置面板，支持填写 `api_base`、`api_key`、测试连接、模型选择与保存。
- 将保存结果写入配置持久化层，并让 LiteLLM Router 在运行时读取该配置。
- 保持与现有静态配置兼容，动态配置不可用时可回退。

**非目标：**
- 不在本次变更中实现多租户配置隔离。
- 不在本次变更中实现密钥托管服务或 KMS 集成。
- 不扩展到 embedding/rerank 全部供应商配置，仅覆盖 LLM 路由主配置。

## 决策

### 决策 1：模型探测采用后端代理 API，而非前端直连
- 选择：前端调用本地 `/api/admin/detect-models`，后端再调用 `${api_base}/models`。
- 原因：可统一处理超时、鉴权、异常映射，规避浏览器跨域限制与 API Key 暴露。
- 备选：前端直连供应商 `/models`。放弃原因：跨域不可控、密钥泄漏风险高、错误处理分散。

### 决策 2：统一模型返回结构为 OpenAI models list 子集
- 选择：后端返回 `{models:[{id,owned_by?}], raw_count}` 的稳定结构，仅保留前端选择所需字段。
- 原因：减少前端与供应商差异耦合，便于后续替换不同兼容供应商。
- 备选：透传上游完整 JSON。放弃原因：结构不稳定，前端耦合上游字段。

### 决策 3：保存配置采用“持久化覆盖 + 静态回退”读取策略
- 选择：Router 先读持久化配置（若有效），否则回退 `configs/default.yaml` 与环境变量。
- 原因：保持向后兼容，可渐进上线。
- 备选：完全替换为动态配置。放弃原因：风险高，回滚困难。

### 决策 4：错误码分层映射
- 选择：区分参数错误、鉴权失败、上游超时、网络不可达、上游响应非法，前端据此展示明确提示。
- 原因：提高可运维性，减少“连接失败”黑盒问题。
- 备选：统一 500 + message。放弃原因：不可诊断。

## 风险 / 权衡

- [风险] API Key 持久化明文风险 → 缓解：限定配置文件访问权限、日志脱敏、禁止在响应中回显 key。
- [风险] 上游兼容接口返回非标准 JSON → 缓解：增加结构校验与 `UPSTREAM_INVALID_RESPONSE` 错误码。
- [风险] 动态配置损坏导致路由不可用 → 缓解：加载失败自动回退静态配置，并记录告警。
- [风险] 前端误操作保存错误配置影响线上 → 缓解：保存前必须通过检测且已选择模型；支持快速重置为默认配置。

## Migration Plan

1. 新增后端检测接口与配置持久化读写实现，保留旧路径不变。
2. 为 Router 增加动态配置读取分支与回退逻辑，默认开启回退。
3. 上线前端设置面板，默认仅管理员可见（若权限体系尚未就绪则先内部环境启用）。
4. 补充接口与前端交互测试，验证鉴权失败、超时、网络错误与保存后重启可用。
5. 如出现回归，关闭动态配置读取开关或清空持久化配置，系统回退静态配置。
