## 新增需求

### 需求:系统必须提供 LLM 连接设置面板
系统必须在前端提供设置面板，至少包含 `API Base` 输入框、`API Key` 输入框与“测试连接并获取模型”按钮。

#### 场景:管理员填写连接参数
- **当** 管理员进入设置面板
- **那么** 系统必须显示 API Base、API Key 和测试按钮

### 需求:系统必须在探测成功后提供模型选择
系统必须在模型探测成功后展示模型下拉框，并禁止在未探测成功时展示无效模型选项。

#### 场景:探测成功
- **当** 后端返回至少一个可用模型
- **那么** 系统必须显示模型下拉并允许管理员选择目标模型

### 需求:系统必须支持保存最终配置
系统必须提供保存按钮，将 `api_base`、`api_key` 与选中模型提交到后端配置持久化接口。

#### 场景:完成配置保存
- **当** 管理员已选择模型并点击保存
- **那么** 系统必须提交最终配置并反馈保存成功状态

### 需求:系统必须提供可理解的错误反馈
系统必须根据后端错误码展示可理解提示，禁止仅显示无上下文的通用失败文案。

#### 场景:鉴权失败提示
- **当** 后端返回 `AUTH_FAILED`
- **那么** 系统必须提示管理员检查 API Key 或权限
