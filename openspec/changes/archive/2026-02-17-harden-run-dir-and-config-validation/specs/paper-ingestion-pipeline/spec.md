## 新增需求

### 需求:chunk 参数运行时校验
系统必须在运行时校验 `chunk_size` 与 `overlap` 配置值，并在非法值出现时应用安全回退，以避免切块行为异常。

#### 场景:chunk_size 越界
- **当** `chunk_size` 小于 300 或大于 500
- **那么** 系统必须记录告警并回退到默认安全值后继续执行

#### 场景:overlap 非法
- **当** `overlap` 小于 0 或大于等于生效的 `chunk_size`
- **那么** 系统必须记录告警并回退到默认安全值后继续执行

## 修改需求

### 需求:分块策略
系统必须优先依据可识别章节标题进行分段，并在分段过长时使用滑窗策略进行切块，目标窗口大小必须落在 300-500 token 近似区间，且相邻窗口必须包含约 50 token 重叠。系统必须在应用配置前完成参数合法性校验，禁止使用越界参数直接执行切块。

#### 场景:超长段落触发滑窗
- **当** 某段文本超过单个 chunk 目标长度
- **那么** 系统必须将其拆分为多个重叠 chunk，且相邻 chunk 保持约 50 token 重叠

#### 场景:参数非法时回退后切块
- **当** 读取到非法 `chunk_size` 或 `overlap` 配置
- **那么** 系统必须先完成回退与告警，再按回退后的参数执行切块
