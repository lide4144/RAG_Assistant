## 为什么

当前实现在同一分钟内多次运行可能复用同一 `runs/YYYYMMDD_HHMM` 目录，存在轨迹文件覆盖风险，削弱可追溯性。同时，`chunk_size/overlap` 缺乏运行时边界校验，配置解析器也较简化，存在配置错误未被及时拦截的稳定性风险。

## 变更内容

- 修改运行目录生成策略，确保每次运行目录唯一，避免同分钟多次运行相互覆盖。
- 为 `chunk_size` 与 `overlap` 增加运行时约束校验与安全回退/告警策略。
- 将配置文件解析从简化行解析升级为标准 YAML 解析方案，提升格式兼容与鲁棒性。
- 保持现有 ingest 输出结构与 CLI 兼容，不引入破坏性接口变更。

## 功能 (Capabilities)

### 新增功能

### 修改功能
- `pipeline-development-conventions`: 调整“每次运行目录”的唯一性语义，并明确配置解析实现要求（标准 YAML 解析）。
- `paper-ingestion-pipeline`: 增加 chunk 参数运行时校验约束，防止越界配置直接生效。

## 影响

- 代码影响：`app/runlog.py`、`app/config.py`、`app/ingest.py` 的运行目录与配置读取路径。
- 依赖影响：可能新增 YAML 解析依赖（如 `PyYAML`）。
- 运行影响：同分钟重复运行将生成不同目录；非法 chunk 配置将触发校验与回退。
- 测试影响：需补充运行目录唯一性与配置校验相关测试用例。
