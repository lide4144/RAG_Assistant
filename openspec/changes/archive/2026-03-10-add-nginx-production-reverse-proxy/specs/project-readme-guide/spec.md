## 修改需求

### 需求:README 排障与验证指引
系统必须在 `README.md` 中提供常见失败场景的排障指引，并且每个场景必须包含“症状、可能原因、检查步骤、修复建议”。对于生产部署，README 还必须提供单域 Nginx 反向代理的最小部署结构、关键路径转发规则和验证步骤；对于 Cloud Studio 等端口封装环境，README 还必须说明单端口代理入口的启动方式。

#### 场景:QA 命令返回空结果或低相关证据
- **当** 用户运行 QA 后证据为空或相关性明显偏低
- **那么** README 必须提供可执行的检查路径（索引文件、graph 文件、配置项、输入数据完整性）与对应修复建议

#### 场景:用户进行生产反代部署
- **当** 用户按照 README 在生产环境部署该项目
- **那么** README 必须说明前端、Kernel、Gateway 的内部监听关系、Nginx 对 `/`、`/api/*`、`/ws` 的代理方式，以及验证部署是否成功的步骤

#### 场景:用户在 Cloud Studio 中部署
- **当** 用户通过 Cloud Studio 的应用封装能力暴露该项目
- **那么** README 必须说明为什么只应暴露单个代理端口，以及对应的启动脚本或操作步骤
