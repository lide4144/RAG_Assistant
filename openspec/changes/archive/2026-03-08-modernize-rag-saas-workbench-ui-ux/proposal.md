## 为什么

当前 RAG SaaS 工作台已具备可用的底层能力，但前端界面仍暴露大量机器语义（底层变量名、原始 JSON、状态枚举直出），导致认知负担高、可读性差、操作信心不足。现在进行 UI/UX 现代化重构，可以在不改变核心流程的前提下显著提升可用性、可维护性与产品化质量。

## 变更内容

- 重构全局布局与状态表达：将顶部降级提示改为可折叠语义化 Alert，连接状态改为非按钮化轻量指示器，并统一修复文案与层级。
- 引入前端状态映射机制：新增全局 `StatusMapper` 约束，禁止页面直接 `JSON.stringify` 输出底层状态对象，所有状态必须先映射为用户可读文案/标签。
- 重构知识处理页：优化统计卡视觉焦点（`tabular-nums` + 放大数值）、将阶段卡重构为横向流水线步骤条、升级日志控制台为终端风格、优化导入记录表的长 ID 截断/提示/复制交互。
- 重构模型设置页：增加“继承全局配置”与“独立配置（Override）”交互，移除底部原始 JSON 回显，引入字段级“未保存”柔和徽章反馈。
- 重构对话问答页：为空会话增加快捷提示卡，顶部模型摘要由长串拼接文本改为结构化语义标签。
- 本次变更不引入破坏性 API（无 **BREAKING** 项）。

## 功能 (Capabilities)

### 新增功能
- `frontend-status-mapping-and-display-governance`: 建立跨页面统一的状态映射与展示治理能力（含禁止原始 JSON 直出、统一状态语义映射与组件化呈现规则）。

### 修改功能
- `frontend-saas-shell-navigation`: 调整全局壳层中的告警条与连接状态指示表达，优化信息分层与可读性。
- `frontend-pipeline-ops-dashboard`: 重构知识处理页的信息架构、步骤可视化、日志终端样式与历史表格交互。
- `frontend-stage-llm-settings`: 增加模型配置继承/覆盖逻辑，移除底部 JSON 回显，增强未保存状态反馈。
- `frontend-chat-focused-experience`: 增强聊天空状态引导与模型摘要语义标签化展示。

## 影响

- 前端代码：`frontend` 下全局布局组件、状态展示组件、知识处理页、模型设置页、聊天页及其样式体系。
- 前端规范资产：需要新增 1 个 capability spec，并对 4 个既有 capability 写增量规范。
- 交互与可用性：页面信息密度、视觉层级、状态可理解性、冷启动引导与可操作性将显著变化。
- 测试与验收：需补充/更新 UI 级回归用例（状态映射、条件渲染、Tooltip/复制行为、未保存徽章、空状态快捷发送）。
