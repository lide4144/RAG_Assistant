## 修改需求

### 需求:系统必须提供全局 SaaS 导航壳层
系统必须在壳层顶部提供语义化运行状态提示区。处于降级态时默认仅展示简版告警文案 `系统当前处于降级运行状态 (DEGRADED)`，并必须提供“查看详情”折叠能力，在展开区以列表展示降级参数明细。系统必须提供“前往模型设置”文字按钮并保持右侧对齐。

#### 场景:降级状态默认展示
- **当** 运行态等级为 `DEGRADED`
- **那么** 页面必须显示折叠态告警条且默认不展开参数明细

#### 场景:展开查看降级详情
- **当** 用户点击“查看详情”
- **那么** 系统必须以结构化列表展示 fallback 参数键值并允许再次折叠

### 需求:系统必须在导航中标记当前页面状态
系统必须将连接状态显示为不可点击的轻量级状态指示器，禁止以按钮样式渲染。连接成功时必须展示绿色发光圆点与文本 `🟢 已连接`。

#### 场景:连接成功指示
- **当** 系统与后端健康检查通过
- **那么** 导航区域必须展示无边框状态指示器而非按钮控件
