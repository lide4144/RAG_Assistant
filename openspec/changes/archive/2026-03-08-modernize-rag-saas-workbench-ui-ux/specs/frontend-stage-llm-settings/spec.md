## 新增需求

### 需求:前端必须提供字段级未保存状态提示
前端必须在用户修改但尚未保存的字段标签旁显示柔和的“⚠️ 未保存”徽章，并必须在保存成功后清除该标记。

#### 场景:修改字段后未保存
- **当** 用户修改任一模型配置字段但未点击保存
- **那么** 对应字段标签必须显示“⚠️ 未保存”提示

## 修改需求

### 需求:前端必须提供三路 Stage LLM 设置面板
系统必须为辅助模型节点提供“继承全局配置”能力。对于 Rewrite、Graph Entity 等辅助节点，默认必须隐藏 `api_base` 与 `api_key` 输入框并显示“🔄 已继承全局配置”。仅当用户显式开启 `独立配置 (Override)` 时，系统才可展开并允许编辑独立凭据字段。

#### 场景:辅助模型默认继承
- **当** 用户进入模型设置页且未开启 Override
- **那么** 辅助模型必须隐藏独立凭据输入并显示继承提示

#### 场景:启用 Override
- **当** 用户开启辅助模型 Override
- **那么** 系统必须展开该模型独立 `api_base` 和 `api_key` 字段

### 需求:前端必须正确回显三路持久化配置
前端在配置回显与展示时必须移除“已保存值/生效值”的原始 JSON 代码块，并仅保留结构化字段视图，禁止向最终用户暴露底层 JSON。

#### 场景:刷新页面后读取配置
- **当** 管理员刷新页面并重新加载配置
- **那么** 系统必须以结构化表单回显配置且不显示底部原始 JSON 区块
