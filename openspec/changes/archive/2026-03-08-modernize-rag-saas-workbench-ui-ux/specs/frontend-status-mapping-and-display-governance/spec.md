## 新增需求

### 需求:前端必须通过统一 StatusMapper 输出用户可读状态
前端必须提供全局 `StatusMapper`，并必须将底层状态值映射为可读文案、语义颜色与图标，禁止在业务组件中直接暴露底层枚举。

#### 场景:映射 Pipeline 阶段状态
- **当** 阶段状态为 `not_started`、`processing`、`unknown`
- **那么** 系统必须分别映射为 `⚪️ 待处理`、`🔵 处理中`、`🟡 未知`

#### 场景:未知状态降级
- **当** 收到未注册状态值
- **那么** 系统必须回退为统一“未知状态”语义并记录可观测日志

### 需求:前端必须禁止直接输出原始 JSON 到页面
前端页面渲染层必须禁止直接 `JSON.stringify` 输出后端原始对象，任何调试信息必须通过受控开发模式开关展示，且默认对最终用户不可见。

#### 场景:生产模式页面渲染
- **当** 页面渲染运行时状态、配置摘要或历史记录
- **那么** 系统必须仅展示经过映射和格式化的语义化信息
