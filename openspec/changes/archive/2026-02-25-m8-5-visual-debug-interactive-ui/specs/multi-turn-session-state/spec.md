## MODIFIED Requirements

### 需求:会话清空与隔离
系统必须提供 `clear_session(session_id)` 接口。调用后，指定会话历史必须被删除；不同 `session_id` 的历史必须隔离，禁止跨会话污染。系统必须支持 UI 层触发该接口；当 UI 执行“开启新对话/清空上下文”后，后续问题必须按空历史会话处理，且不得使用清空前会话中的脱水记录、实体集合或上一轮决策信号。

#### 场景:清空后重新提问
- **当** 前端调用 `clear_session(session_id)` 后用户发起新问题
- **那么** 系统必须按新会话起点处理，`history_used_turns` 必须为 0 或等效空历史状态

#### 场景:UI 触发清空后无历史污染
- **当** 用户在 UI 点击“开启新对话/清空上下文”并发送与上一会话无关的问题
- **那么** 系统在 rewrite 与检索阶段必须仅读取新会话历史，禁止引入清空前会话的上下文

## ADDED Requirements

### 需求:系统必须输出会话重置信号供 UI 审查
系统必须在运行 trace 中提供会话重置结果字段（例如 `session_reset=true/false` 或等价字段），用于确认清空动作已在后端生效。

#### 场景:清空动作可观测
- **当** 本轮请求由 UI 清空会话后发起
- **那么** run trace 必须包含会话已重置的可序列化信号
