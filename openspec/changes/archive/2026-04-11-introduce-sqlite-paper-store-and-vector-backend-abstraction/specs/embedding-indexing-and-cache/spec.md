## 新增需求

### 需求:Embedding 索引构建必须通过统一向量后端接口执行
Embedding 索引构建、删除、重建与检索前的后端准备逻辑必须通过统一向量后端接口执行。系统禁止让 embedding 流程直接依赖某个具体后端的文件布局或未来 Qdrant SDK 细节。

#### 场景:构建文件向量后端索引
- **当** 当前环境选择文件向量后端
- **那么** embedding 索引构建流程必须通过统一后端接口执行写入，并由文件后端实现具体落盘行为

#### 场景:未来切换到其他向量后端
- **当** 管理员切换到未来的 Qdrant 或其他后端实现
- **那么** embedding 流程必须继续通过同一后端接口执行，而不是修改调用方业务语义

### 需求:Embedding 索引元信息必须回写权威存储
Embedding 索引构建完成后，系统必须将当前向量后端类型、模型信息、构建时间、适用语料范围或快照标识、是否过期等元信息写入权威存储。系统禁止仅把这些信息写在文件头或运行日志中。

#### 场景:查询当前 embedding 后端状态
- **当** 用户或后台任务查询当前 embedding 索引状态
- **那么** 系统必须返回当前后端类型、索引元信息与是否需要重建

#### 场景:单篇论文引发局部过期
- **当** 某篇论文的 chunk 内容变化导致 embedding 语料过期
- **那么** 系统必须能够将相关索引状态标记为局部或整体过期，并保留对应后端元信息
