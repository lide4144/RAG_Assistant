## 新增需求

### 需求:批量导入界面必须展示与论文实体一致的逐篇状态
批量导入界面必须展示与 SQLite 论文生命周期一致的逐篇状态，而不是仅展示上传文件名对应的临时进度项。系统禁止在页面刷新、任务结束或会话切换后丢失单篇论文状态可见性。

#### 场景:页面刷新后恢复逐篇状态
- **当** 用户在导入任务运行期间刷新页面或稍后重新进入导入页
- **那么** 系统必须能够基于 SQLite 论文记录与任务状态恢复该批论文的逐篇状态展示

#### 场景:同一批导入包含新增与重复论文
- **当** 一次批量导入中同时存在新增论文与已存在论文
- **那么** 页面必须分别展示新增、重复跳过与失败的逐篇状态，并与对应 `paper_id` 或稳定记录一致

### 需求:导入结果必须能够指向论文级后续动作
导入结果必须允许用户基于单篇论文状态继续执行查看详情、删除、重建或进入聊天验证等后续动作。系统禁止仅提供整批任务级结果而无法定位到具体论文。

#### 场景:单篇论文索引失败后重建
- **当** 某篇论文导入成功但索引阶段失败
- **那么** 页面必须能够定位该论文并提供重建索引或等价后续动作

#### 场景:单篇论文已 ready
- **当** 某篇论文已完成处理并进入 `ready`
- **那么** 页面必须能够从导入结果直接跳转到该论文相关的目录详情或聊天验证入口
